# SRT Flow 项目管理模式文档

## 1. 概述

为了支持多个 AI Coding Agent 协同开发，本项目采用 **“Markdown 看板 + 任务文件”** 的去中心化管理模式。
所有状态均持久化在文件系统中，Agent 通过读取和修改特定文件来同步进度，避免冲突。

## 2. 核心文件结构

```text
srt-flow/
├── KANBAN.md           # 全局看板 (任务状态总览)
└── tasks/              # 任务详情目录
    ├── TASK-001.md     # 具体任务描述文件
    ├── TASK-002.md
    └── ...
```

## 3. 看板设计 (KANBAN.md)

`KANBAN.md` 是项目进度的唯一真理来源。它包含四个主要区域（泳道）。

### 格式示例

```markdown
# Project Kanban

## 📋 Todo (待办)
- [ ] [TASK-003] 编写 Downloader Service 基础类
- [ ] [TASK-004] 设计数据库模型

## 🏗️ In Progress (进行中)
- [ ] [TASK-001] 初始化项目结构 (@Agent-A)
- [ ] [TASK-002] 配置 Docker 环境 (@Agent-B)

## 👀 Review (待审查)
- [ ] [TASK-000] 需求分析 (@Agent-A)

## ✅ Done (已完成)
- [x] [TASK-XXX] ...
```

**规则**：
1.  Agent **必须** 先在 `KANBAN.md` 中将任务从 `Todo` 移动到 `In Progress` 并署名，才能开始工作。
2.  同一时间，一个 Agent 只能处于一个任务的 `In Progress` 状态。
3.  如果发现 `In Progress` 中已有其他 Agent 署名，**严禁** 修改该任务相关代码，防止冲突。

## 4. 任务文件设计 (tasks/TASK-XXX.md)

每个任务对应一个独立的 Markdown 文件，包含详细的上下文和执行标准。

### 模板

```markdown
---
id: TASK-001
title: 初始化项目结构
status: In Progress
assignee: Agent-A
priority: High
---

# 任务描述
搭建 FastAPI + Vue.js 的基础目录结构，配置 poetry/pip 依赖。

# 验收标准 (Acceptance Criteria)
- [ ] 后端 FastAPI 能成功启动并响应 /health 接口
- [ ] 前端 Vue 能成功 build 并显示 Hello World
- [ ] 提交 .gitignore 文件

# 上下文 (Context)
- 参考 `系统架构文档.md` 中的目录规划
- 忽略 `data/` 目录
```

## 5. 协作工作流 (Workflow)

### 阶段 1: 领取任务 (Claim)
1.  Agent 读取 `KANBAN.md`。
2.  找到 `Todo` 列表中优先级最高的任务。
3.  **原子操作**：
    *   将该行移动到 `In Progress`。
    *   在行尾添加 `(@Agent-Name)`。
    *   修改对应的 `tasks/TASK-XXX.md`，更新 `status: In Progress` 和 `assignee`。

### 阶段 2: 执行任务 (Execute)
1.  Agent 阅读 `tasks/TASK-XXX.md` 理解需求。
2.  编写代码。
3.  每完成一个小项，更新 `tasks/TASK-XXX.md` 中的 Checkbox。

### 阶段 3: 提交审查 (Submit)
1.  Agent 确保所有代码通过测试。
2.  **原子操作**：
    *   在 `KANBAN.md` 中将任务移动到 `Review`。
    *   更新 `tasks/TASK-XXX.md`，`status: Review`。
3.  (可选) 生成 `Review Request` 通知用户。

### 阶段 4: 完成 (Done)
1.  用户或 Lead Agent 审查通过。
2.  将任务移动到 `Done`，并标记为 `[x]`。

## 6. 冲突避让机制

*   **文件锁模拟**：`KANBAN.md` 中的 `In Progress` 栏即为“软锁”。
*   **模块隔离**：在架构设计阶段已强调模块化，尽量让不同 Task 修改不同目录下的文件。
*   **频繁同步**：Agent 在开始任何写操作前，建议重新读取 `KANBAN.md` 确认状态未被篡改。
