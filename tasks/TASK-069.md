# TASK-069: 优化翻译服务空行检测策略

## 任务信息
- 优先级: 高
- 状态: Todo
- 来源: 生产问题
- 相关功能: 翻译服务

## 问题描述

翻译服务在处理某些视频字幕时，频繁出现"空行比例超过阈值"的错误，导致所有翻译实例失败：

```
All instances failed: iFLow心流开放平台: Empty line ratio 23.3% exceeds threshold 10.0% (7/30 empty lines) (after 2 empty line retries)
```

### 问题分析

1. **当前机制**：
   - 默认空行阈值：30%（配置中为 0.3）
   - 错误信息显示阈值：10%（与配置不符）
   - 空行重试次数：2 次
   - 超过阈值后会重试，重试失败则整个批次失败

2. **可能原因**：
   - AI 模型对某些内容返回空翻译
   - 批次大小设置不合理
   - 阈值设置过于严格
   - 某些字幕本身就应该为空（如纯音效、背景音乐标记）

3. **影响**：
   - 翻译任务失败率高
   - 用户体验差
   - 需要手动重试多次

## 优化方案

### 方案一：调整阈值和重试策略（推荐）

1. **提高默认阈值**：
   - 从 30% 提高到 50%
   - 允许更多空行存在

2. **增加重试次数**：
   - 从 2 次增加到 3 次
   - 给 AI 更多机会纠正

3. **智能阈值**：
   - 根据批次大小动态调整阈值
   - 小批次（< 10 条）：允许更高的空行比例
   - 大批次（≥ 30 条）：使用标准阈值

### 方案二：空行后处理

1. **保留空行**：
   - 不再将空行视为错误
   - 在后续处理中智能填充或保留

2. **智能填充**：
   - 检测空行的上下文
   - 如果是音效标记（如 [Music]、[Applause]），保留原文
   - 如果是普通字幕，尝试重新翻译单条

3. **用户选择**：
   - 提供配置选项：严格模式 vs 宽松模式
   - 严格模式：保持当前逻辑
   - 宽松模式：允许空行，后续处理

### 方案三：改进提示词

1. **明确指示**：
   - 在提示词中明确要求：即使无法翻译，也要返回原文
   - 强调：不要返回空行

2. **示例引导**：
   - 在提示词中添加示例，展示如何处理特殊内容

## 实现内容

### 阶段一：快速修复（推荐先实施）

1. **调整默认配置**：
   - `backend/core/config.py` - TranslatorConfig
     - `empty_line_threshold`: 0.3 → 0.5
   
2. **增加重试次数**：
   - `backend/services/translator.py` - BatchTranslator
     - `EMPTY_LINE_MAX_RETRIES`: 2 → 3

3. **修复阈值显示不一致**：
   - 检查为什么错误信息显示 10% 而配置是 30%
   - 确保显示的阈值与实际配置一致

### 阶段二：智能优化

1. **动态阈值**：
   ```python
   def _get_dynamic_threshold(self, batch_size: int) -> float:
       if batch_size < 10:
           return 0.6  # 小批次允许 60%
       elif batch_size < 20:
           return 0.5  # 中批次允许 50%
       else:
           return self._empty_line_threshold  # 大批次使用配置值
   ```

2. **空行分类**：
   - 区分"完全空行"和"音效标记"
   - 音效标记不计入空行统计

3. **改进提示词**：
   - 在 system prompt 中添加：
     ```
     重要：即使某些内容难以翻译，也必须返回原文或合理的翻译。
     不要返回空行或仅包含序号的行。
     对于音效标记（如 [Music]、[Applause]），保持原样。
     ```

### 阶段三：配置化

1. **前端配置项**：
   - 添加"空行处理策略"选项
   - 严格模式 / 标准模式 / 宽松模式

2. **批次大小优化**：
   - 根据空行率动态调整批次大小
   - 如果连续失败，自动减小批次

### 需要注意

1. **向后兼容**：
   - 修改默认值不影响已有配置
   - 用户可以在配置中覆盖

2. **日志记录**：
   - 记录空行检测的详细信息
   - 便于分析和调试

3. **性能影响**：
   - 增加重试次数会增加翻译时间
   - 需要权衡准确性和速度
