# TASK-068: 语音合成句子合并分批处理优化

## 任务信息
- 优先级: 中
- 状态: Todo
- 来源: 技术优化
- 相关任务: TASK-057
- 相关功能: 语音合成-句子合并

## 问题描述

当前的句子合并功能（TASK-057）是一次性将所有字幕条目发送给 AI 处理，存在以下问题：

1. **Token 限制风险**：超长视频（5000+ 条字幕）可能超出 LLM 的上下文窗口（128K tokens）
2. **响应超时风险**：大量字幕一次性处理可能导致 API 超时
3. **进度不可见**：用户无法看到处理进度，体验不佳

## 优化方案

采用**两阶段法**进行分批处理：

### 第一阶段：AI 判断合并关系

- 将所有字幕条目的文本发送给 AI
- AI 只返回"哪些条目应该合并"，不返回合并后的文本
- 输出格式极简：`1,2,3|4,5|6|7,8,9` 表示合并分组
- 优势：输出 token 极少，不会超长

### 第二阶段：本地分批处理

- 根据 AI 返回的合并关系，在本地将字幕条目分组
- 每组内的字幕文本拼接起来
- 分批进行 TTS 合成（纯技术性分批，不涉及语义判断）

### 超长字幕处理

如果字幕条目 > 4000 条（保守阈值）：
- 在标点符号处切分成多段（每段 ≤ 4000 条）
- 对每段独立调用 AI 判断合并关系
- 本地汇总所有合并关系后，统一分批 TTS

## 技术要点

### 核心优势

1. **彻底避免批次衔接问题**：AI 一次性看到完整上下文，不存在跨批次断句
2. **输出极简**：只返回数字分组，token 消耗极少
3. **逻辑简单**：本地分批是纯机械操作，不会出错
4. **结果准确**：AI 能看到所有上下文，合并判断最准确

### Token 容量分析

- DeepSeek V3/R1 上下文：128K tokens
- 单条字幕平均：约 20 tokens（含序号和格式）
- 可处理字幕数：128K ÷ 20 = 约 6,400 条
- 保守阈值：4,000 条（预留 system prompt 和输出空间）

### 实际场景

- 1 小时视频：约 800-1,200 条字幕 ✅ 一次处理
- 2 小时视频：约 1,600-2,400 条字幕 ✅ 一次处理
- 4 小时视频：约 3,200-4,800 条字幕 ⚠️ 可能需要切分
- 8 小时视频：约 6,400-9,600 条字幕 ⚠️ 需要切分

## 实现内容

### 后端修改

1. **`backend/services/tts.py` - SentenceMerger 类重构**
   
   新增方法：
   - `merge_with_batching()`: 主入口，自动判断是否需要分批
   - `_merge_single_batch()`: 单批次处理（< 4000 条）
   - `_merge_multiple_batches()`: 多批次处理（≥ 4000 条）
   - `_split_at_punctuation()`: 在标点符号处智能切分
   - `_parse_merge_groups()`: 解析 AI 返回的分组关系
   
   修改方法：
   - `merge()`: 调用新的 `merge_with_batching()`
   - `_call_api()`: 支持新的提示词格式（只返回分组）
   - `_parse_response()`: 解析新的输出格式（数字分组）

2. **提示词优化**
   
   新的 system prompt：
   ```
   你是一个字幕句子合并助手。你的任务是判断哪些相邻的字幕片段属于同一个完整句子。
   
   规则：
   1. 分析输入的字幕列表，识别哪些相邻片段属于同一句子
   2. 只输出合并分组，不要输出合并后的文本
   3. 输出格式：用竖线分隔不同的组，组内用逗号分隔序号
   4. 示例：1,2,3|4,5|6|7,8,9 表示第1-3条合并、第4-5条合并、第6条独立、第7-9条合并
   5. 不要添加任何解释，只输出分组结果
   ```

3. **进度报告优化**
   
   - 第一阶段：报告 AI 判断进度（如果有多批次）
   - 第二阶段：报告 TTS 合成进度（按组计算）

### 配置项（可选）

在 TTSConfig 中新增：
- `sentence_merge_batch_size`: 单批次最大字幕数（默认 4000）
- `sentence_merge_split_at_punctuation`: 是否在标点符号处切分（默认 true）

### 测试场景

1. **短视频**（< 1000 条字幕）：一次性处理，验证结果正确性
2. **中等视频**（1000-3000 条字幕）：一次性处理，验证性能
3. **长视频**（4000-6000 条字幕）：分批处理，验证切分逻辑
4. **超长视频**（> 6000 条字幕）：多批次处理，验证结果衔接

## 实现总结

已完成 SentenceMerger 类的重构，实现两阶段分批处理：

### 核心改进

1. **新的提示词格式**：AI 只返回分组关系（如 `1,2,3|4,5|6`），不返回合并后的文本
2. **自动分批逻辑**：
   - < 4000 条：单批次处理
   - ≥ 4000 条：自动分批，在标点符号处智能切分
3. **两阶段处理**：
   - 阶段 1：AI 判断合并关系
   - 阶段 2：本地根据分组关系合并文本
4. **增强的容错性**：
   - 处理缺失的索引
   - 处理重复的索引
   - 解析失败时自动补全

### 代码变更

**backend/services/tts.py - SentenceMerger 类**：
- 新增 `DEFAULT_BATCH_SIZE = 4000` 常量
- 更新 `DEFAULT_SYSTEM_PROMPT` 和 `DEFAULT_USER_PROMPT`
- 重构 `merge()` 方法：自动判断是否需要分批
- 新增 `_get_merge_groups_single()`：单批次处理
- 新增 `_get_merge_groups_batched()`：多批次处理
- 新增 `_split_into_batches()`：智能切分批次
- 新增 `_parse_merge_groups()`：解析分组关系
- 新增 `_build_merged_entries()`：构建合并结果
- 更新 `_call_api()`：增加 max_tokens 和 timeout
- 删除旧的 `_parse_response()` 方法

### 向后兼容

- 所有现有调用无需修改（batch_size 参数可选）
- TTSService 和 routers/tts.py 中的调用保持兼容
