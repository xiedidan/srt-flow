# TASK-059: 语音合成句子合并预览与调整

## 任务信息
- 优先级: 中
- 状态: Review
- 来源: 需求补充
- 相关功能: 语音合成
- 依赖任务: TASK-057

## 需求描述

TASK-057 实现了语音合成前的 AI 句子合并功能，但目前合并是自动执行的，用户无法预览和调整合并结果。需要在语音合成任务创建流程中增加句子合并预览和调整功能。

## 功能要求

1. **合并预览**
   - 在创建语音合成任务时，如果启用了句子合并，先执行 AI 合并
   - 显示合并前后的对比视图
   - 清晰展示哪些字幕被合并到了一起

2. **手动调整**
   - 用户可以手动拆分已合并的句子
   - 用户可以手动合并相邻的句子
   - 用户可以编辑合并后的文本内容

3. **确认提交**
   - 用户确认合并结果后，再提交语音合成任务
   - 将调整后的合并结果传递给 TTS 服务

## 技术要点

### 后端
- 新增 API：执行句子合并预览（不执行 TTS，只返回合并结果）
- 修改 TTS 任务：支持接收用户调整后的合并数据

### 前端
- 修改 `TtsTaskCreateModal.vue`：
  - 增加句子合并预览步骤
  - 实现合并结果编辑界面
  - 支持拆分/合并/编辑操作
- 新增组件：`SentenceMergeEditor.vue`
  - 显示原始字幕和合并后字幕的对比
  - 提供拖拽或按钮操作进行调整

## 实现内容

### 后端修改

1. `backend/app/routers/tts.py` - 新增 TTS 路由：
   - `POST /tts/sentence-merge/preview`: 预览 AI 句子合并结果
   - `POST /tts/sentence-merge/adjust`: 手动调整合并条目（合并/拆分/编辑）

2. `backend/services/tts.py` - TTSService.execute() 修改：
   - 新增 `skip_ai_merge` 参数支持跳过 AI 合并
   - 新增 `merged_entries` 参数接收用户调整后的合并数据
   - 当用户提供合并数据时直接使用，不再调用 AI

3. `backend/app/routers/__init__.py` - 注册 TTS 路由

### 前端修改

1. `frontend/src/api/tts.js` - 新增 TTS API 模块：
   - `previewSentenceMerge()`: 调用句子合并预览 API
   - `adjustMergedEntries()`: 调用调整合并条目 API

2. `frontend/src/components/tts/SentenceMergeEditor.vue` - 新增组件：
   - 显示原始字幕和合并后字幕的对比
   - 支持选中条目进行操作
   - 提供合并上/合并下/拆分/编辑按钮
   - 编辑弹窗和拆分弹窗
   - 重新生成按钮

3. `frontend/src/components/tts/TtsTaskCreateModal.vue` - 重构为多步骤流程：
   - 步骤 1: 选择视频和配置选项
   - 步骤 2: 句子合并预览（仅单视频且启用合并时显示）
   - 步骤指示器显示当前进度
   - 批量任务自动跳过预览步骤
   - 提交时传递用户调整后的合并数据
