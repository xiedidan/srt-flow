# TASK-070: 修复模态框关闭逻辑

## 任务描述

修复模态框的关闭逻辑问题。当前在模态框内部按下鼠标，然后移动到模态框外部释放鼠标时，会导致模态框关闭。这造成很难选中模态框内部的文字，必须严格在模态框内部释放鼠标。

## 问题分析

现有问题：
- 在模态框内部按下鼠标 → 移动到外部释放 → 模态框关闭
- 导致无法正常选中模态框内的文字

## 需求

模态框应该只在以下两种情况下关闭：
1. 在模态框外部按下鼠标（释放鼠标不会关闭）
2. 点击模态框的关闭按钮

## 实现方案

### 前端修改

1. **修改关闭逻辑**
   - 监听 `mousedown` 事件而非 `click` 事件
   - 记录 `mousedown` 发生的位置
   - 只有当 `mousedown` 发生在模态框外部时才关闭

2. **涉及的组件**
   - 所有使用模态框的组件（视频管理、任务管理等）
   - 可能需要创建统一的模态框组件或 composable

### 技术细节

```javascript
// 示例实现
const handleBackdropMouseDown = (e) => {
  if (e.target === e.currentTarget) {
    // 只有点击背景层本身才关闭
    closeModal()
  }
}
```

## 验收标准

- [ ] 在模态框内按下鼠标，移动到外部释放，模态框不关闭
- [ ] 在模态框外部按下鼠标，模态框立即关闭
- [ ] 点击关闭按钮，模态框正常关闭
- [ ] 可以正常选中模态框内的文字
- [ ] 所有使用模态框的页面都应用了新逻辑
