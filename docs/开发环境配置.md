# 开发环境配置指南

本文档详细说明如何配置 SRT Flow 的本地开发环境。

## 系统要求

- Python 3.12+
- Node.js 20+
- Git
- **7-Zip**（必需，用于解压引擎文件）

### 安装 7-Zip

#### Windows

1. 下载安装包：https://www.7-zip.org/download.html
2. 选择 64-bit 版本（推荐）
3. 默认安装到 `C:\Program Files\7-Zip\`
4. **添加到 PATH 环境变量**：
   - 右键点击「此电脑」→「属性」
   - 点击「高级系统设置」
   - 点击「环境变量」
   - 在「系统变量」中找到 `Path`，点击「编辑」
   - 点击「新建」，添加：`C:\Program Files\7-Zip\`
   - 点击「确定」保存所有对话框
5. **验证安装**：
   ```cmd
   # 打开新的命令行窗口
   7z
   # 应该显示 7-Zip 的版本信息和使用说明
   ```

> **重要**：必须打开新的命令行窗口才能使 PATH 更改生效。

#### Linux

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install p7zip-full

# CentOS/RHEL
sudo yum install p7zip p7zip-plugins

# 验证安装
7z
```

#### macOS

```bash
# 使用 Homebrew
brew install p7zip

# 验证安装
7z
```

## 快速开始

### 1. 克隆项目

```bash
git clone https://github.com/your-repo/srt-flow.git
cd srt-flow
```

### 2. 后端环境配置

#### 创建虚拟环境

```bash
# Linux/macOS
python3 -m venv venv
source venv/bin/activate

# Windows
python -m venv venv
venv\Scripts\activate
```

#### 安装依赖

```bash
pip install -r requirements.txt
```

#### 配置环境变量

```bash
# 复制示例配置
cp .env.example .env

# 生成密钥
python -c "import secrets; print('SECRET_KEY=' + secrets.token_hex(32))" >> .env
python -c "import secrets; print('ENCRYPTION_KEY=' + secrets.token_hex(32))" >> .env
```

### 3. 前端环境配置

```bash
cd frontend
npm install
cd ..
```

### 4. 启动开发服务器

#### 使用启动脚本（推荐）

**Linux/macOS:**

```bash
# 添加执行权限（首次运行）
chmod +x scripts/*.sh

# 启动后端（终端 1）
./scripts/dev-backend.sh

# 启动前端（终端 2）
./scripts/dev-frontend.sh
```

**Windows:**

```cmd
REM 启动后端（终端 1）
scripts\dev-backend.bat

REM 启动前端（终端 2）
scripts\dev-frontend.bat
```

#### 手动启动

**后端:**

```bash
# 确保虚拟环境已激活
uvicorn backend.app.main:app --reload --reload-dir backend --host 0.0.0.0 --port 8010
```

**前端:**

```bash
cd frontend
npm run dev
```

### 5. 访问应用

- **前端开发服务器**: http://localhost:5173
- **后端 API**: http://localhost:8010
- **API 文档**: http://localhost:8010/docs
- **ReDoc 文档**: http://localhost:8010/redoc

## 热更新说明

### 后端热更新

后端使用 `uvicorn` 的 `--reload` 模式，会监听 `backend/` 目录下的文件变化：

- **监听范围**: `backend/` 目录下的所有 `.py` 文件
- **触发条件**: 保存文件后自动重启服务
- **重启时间**: 通常 1-2 秒
- **日志输出**: 控制台会显示 "Reloading..." 信息

**注意事项:**

1. 修改 `.env` 文件需要手动重启服务
2. 修改 `requirements.txt` 后需要重新安装依赖
3. 数据库模型变更可能需要手动迁移

### 前端热更新

前端使用 Vite 的 HMR (Hot Module Replacement)：

- **监听范围**: `frontend/src/` 目录下的所有文件
- **触发条件**: 保存文件后自动更新
- **更新方式**: 
  - Vue 组件：热替换，保持状态
  - CSS/SCSS：即时更新样式
  - 其他文件：刷新页面
- **更新时间**: 通常 < 1 秒

**注意事项:**

1. 修改 `vite.config.js` 需要重启开发服务器
2. 修改 `package.json` 后需要重新安装依赖
3. 某些情况下可能需要手动刷新浏览器

## 常见问题

### Q: 后端修改代码后没有自动重启？

A: 检查以下几点：
1. 确保使用了 `--reload` 参数
2. 确保修改的文件在 `backend/` 目录下
3. 检查控制台是否有错误信息
4. 尝试手动重启服务

### Q: 前端修改代码后没有自动更新？

A: 检查以下几点：
1. 确保 Vite 开发服务器正在运行
2. 检查浏览器控制台是否有错误
3. 尝试手动刷新浏览器
4. 清除浏览器缓存

### Q: 虚拟环境激活失败？

A: 
- **Linux/macOS**: 确保有执行权限 `chmod +x venv/bin/activate`
- **Windows**: 可能需要修改执行策略 `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`

### Q: 端口被占用？

A: 
```bash
# 查找占用端口的进程
# Linux/macOS
lsof -i :8010
lsof -i :5173

# Windows
netstat -ano | findstr :8010
netstat -ano | findstr :5173

# 结束进程
kill -9 <PID>  # Linux/macOS
taskkill /PID <PID> /F  # Windows
```

### Q: 依赖安装失败？

A:
```bash
# 清理缓存重试
pip cache purge
pip install -r requirements.txt

# 或使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q: 数据库迁移问题？

A:
```bash
# 删除数据库重新初始化（开发环境）
rm data/srtflow.db
# 重启后端服务会自动创建新数据库
```
