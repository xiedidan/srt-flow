# AI 配置重构设计文档

## 1. 背景与目标

### 1.1 当前问题

当前 AI 配置分散在各服务配置中：
- 翻译服务：`provider`、`model_name`、`base_url`、`api_key`、`temperature`
- 素材生成服务：复用翻译服务配置
- ASR 服务（Gemini）：`gemini_api_key`、`gemini_base_url`

问题：
1. 用户无法配置多个同类型的 API（如多个 OpenAI 账号）
2. API 配置与服务参数耦合，不够灵活
3. 新增 AI 服务时需要重复配置 API

### 1.2 重构目标

1. **API 配置独立**：用户可自由配置多个 AI 提供商
2. **服务引用 API**：各服务选择已配置的 API，搭配自己的参数
3. **向后兼容**：平滑迁移现有配置

## 2. 数据结构设计

### 2.1 AI Provider 配置（新增）

存储在 SQLite `ai_providers` 表：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | UUID 主键 |
| name | VARCHAR(100) | 用户自定义名称（如"我的 DeepSeek"） |
| api_type | VARCHAR(50) | API 类型：deepseek/openai/gemini/openai_compatible |
| base_url | TEXT | API 地址（可选，有默认值） |
| api_key | TEXT | 加密存储的 API Key |
| is_enabled | BOOLEAN | 是否启用 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

### 2.2 API 类型与默认 URL

| API 类型 | 默认 Base URL |
|----------|---------------|
| deepseek | https://api.deepseek.com/v1 |
| openai | https://api.openai.com/v1 |
| gemini | https://generativelanguage.googleapis.com/v1beta |
| openai_compatible | （必填） |

### 2.3 服务配置调整

#### 翻译服务配置（TranslateConfig）

```python
class TranslateConfig(BaseModel):
    # API 选择
    ai_provider_id: Optional[str] = None  # 关联 ai_providers.id
    model_name: str = "deepseek-chat"
    
    # 翻译参数
    batch_size: int = 30
    context_overlap_lines: int = 3
    temperature: float = 1.0
    max_retries: int = 3
    
    # 自定义提示词
    system_prompt: Optional[str] = None
    user_prompt_template: Optional[str] = None
```

#### ASR 服务配置（ASRConfig）

```python
class ASRConfig(BaseModel):
    engine: ASREngine = ASREngine.FASTER_WHISPER_XXL
    
    # Whisper 本地配置
    whisper_model_size: WhisperModelSize = WhisperModelSize.LARGE_V3
    whisper_extra_args: str = ""
    device: ComputeDevice = ComputeDevice.AUTO
    
    # Gemini API 配置（当 engine=gemini 时使用）
    gemini_provider_id: Optional[str] = None  # 关联 ai_providers.id
    gemini_model: str = "gemini-1.5-flash"
```

#### 素材生成服务配置（AssetConfig）

素材生成包含3个子任务，每个子任务有独立的 AI 配置：

```python
class AssetConfig(BaseModel):
    # 功能开关
    generate_thumbnail: bool = True
    generate_summary: bool = True
    generate_titles: bool = True
    
    # 其他配置
    thumbnail_method: str = "extract"
    title_count: int = 3
    
    # 标题生成 AI 配置
    title_ai_provider_id: Optional[str] = None  # 关联 ai_providers.id
    title_model_name: str = "deepseek-chat"
    title_temperature: float = 0.85
    title_system_prompt: Optional[str] = None
    title_user_prompt_template: Optional[str] = None
    
    # 摘要生成 AI 配置
    summary_ai_provider_id: Optional[str] = None  # 关联 ai_providers.id
    summary_model_name: str = "deepseek-chat"
    summary_temperature: float = 0.65
    summary_system_prompt: Optional[str] = None
    summary_user_prompt_template: Optional[str] = None
    
    # 缩略图生成配置（AI 模式时使用）
    # 文本模型配置 - 用于生成文生图提示词
    thumbnail_ai_provider_id: Optional[str] = None  # 关联 ai_providers.id
    thumbnail_model_name: str = "deepseek-chat"
    thumbnail_temperature: float = 0.7
    thumbnail_system_prompt: Optional[str] = None
    thumbnail_user_prompt_template: Optional[str] = None
    
    # 图像生成 API 配置 - 用于调用文生图服务
    thumbnail_image_provider: str = "dall-e"  # dall-e, midjourney, stable-diffusion
    thumbnail_image_api_key: Optional[str] = None  # 加密存储
    thumbnail_image_base_url: Optional[str] = None
    thumbnail_image_model: str = "dall-e-3"
```

**设计理由**：
1. 三个子任务的创作性质不同，需要不同的温度参数
2. 用户可能希望使用不同的模型（如标题用 GPT-4，摘要用 DeepSeek）
3. 缩略图 AI 生成需要两步：先用文本模型生成提示词，再调用图像生成 API

## 3. API 设计

### 3.1 AI Provider 管理 API

| 端点 | 方法 | 说明 |
|------|------|------|
| /api/v1/ai-providers | GET | 获取所有 AI Provider（API Key 脱敏） |
| /api/v1/ai-providers | POST | 创建 AI Provider |
| /api/v1/ai-providers/{id} | GET | 获取单个 AI Provider |
| /api/v1/ai-providers/{id} | PUT | 更新 AI Provider |
| /api/v1/ai-providers/{id} | DELETE | 删除 AI Provider |
| /api/v1/ai-providers/{id}/test | POST | 测试 API 连接 |

#### 创建/更新请求体

```json
{
  "name": "我的 DeepSeek",
  "api_type": "deepseek",
  "base_url": null,
  "api_key": "sk-xxx"
}
```

#### 响应体（列表）

```json
{
  "code": 0,
  "data": [
    {
      "id": "uuid-xxx",
      "name": "我的 DeepSeek",
      "api_type": "deepseek",
      "base_url": "https://api.deepseek.com/v1",
      "api_key_masked": "sk-a***",
      "api_key_configured": true,
      "is_enabled": true,
      "created_at": "2024-12-26T10:00:00Z"
    }
  ]
}
```

### 3.2 服务配置 API 调整

现有配置 API 保持不变，但配置内容调整为引用 `ai_provider_id`。

## 4. 前端设计

### 4.1 设置页面结构调整

```
设置
├── 全局配置
├── AI 服务商 ← 新增
│   ├── [列表] 已配置的服务商
│   └── [按钮] 添加服务商
├── 下载配置
├── 语音识别配置
│   └── Gemini API → 选择已配置的服务商
├── 翻译配置
│   └── AI 服务商 → 选择已配置的服务商
├── 语音合成配置
├── 视频合成配置
└── 素材生成配置
    └── AI 服务商 → 选择已配置的服务商
```

### 4.2 AI 服务商管理界面

- 卡片式列表展示已配置的服务商
- 每个卡片显示：名称、类型、URL（脱敏）、状态
- 操作：编辑、测试连接、删除
- 添加服务商弹窗：
  - 名称（必填）
  - API 类型（下拉选择）
  - 自定义 URL（可选，根据类型显示默认值）
  - API Key（必填）

### 4.3 服务配置中的 AI 选择

使用下拉选择器，选项来自已配置的 AI 服务商列表：
- 显示格式：`{name} ({api_type})`
- 未配置时提示：请先在"AI 服务商"中添加配置

## 5. 迁移方案

### 5.1 数据迁移

1. 检查现有翻译配置中的 `api_key`
2. 如果存在，自动创建对应的 AI Provider
3. 更新翻译配置，设置 `ai_provider_id`
4. 清理旧的 `api_key` 字段

### 5.2 代码迁移

1. 新增 `AIProvider` 模型和 Repository
2. 新增 AI Provider API 路由
3. 修改各服务的配置加载逻辑
4. 修改前端设置页面
