# 视频合成服务详细设计文档

## 1. 模块概述

### 1.1 功能定位

视频合成服务负责将翻译后的字幕烧录到原视频中生成最终输出视频，是 SRT Flow 处理流水线的最后环节。它封装了 FFmpeg 命令，提供字幕烧录功能。

### 1.2 核心职责

- **字幕烧录**：将字幕硬编码到视频画面中
- **双字幕叠加**：支持同时显示原始字幕和翻译字幕
- **样式配置**：支持可配置的字幕样式，双字幕可分别配置不同样式
- **进度上报**：解析 FFmpeg 输出，实时报告合成进度

### 1.3 字幕模式

| 模式 | 说明 |
|------|------|
| 仅原始字幕 | 只烧录原始语言字幕 |
| 仅翻译字幕 | 只烧录翻译后字幕 |
| 双字幕叠加 | 同时显示原始字幕和翻译字幕 |

## 2. 业务流程

### 2.1 合成流程

```
接收合成任务
    │
    ▼
检查输入文件是否存在
    │
    ├── 视频文件不存在 ──→ 返回错误
    │
    ▼
检查字幕文件是否存在
    │
    ├── 字幕文件不存在 ──→ 返回错误
    │
    ▼
构建 FFmpeg 命令
    │
    ▼
执行 FFmpeg 合成
    │
    ▼
解析进度输出
    │
    ▼
验证输出文件
    │
    ▼
更新数据库记录
    │
    ▼
返回合成结果
```

### 2.2 输出文件

合成完成后在视频目录生成 `video_output.mp4`。

## 3. FFmpeg 命令封装

### 3.1 基础命令结构

FFmpeg 命令由以下部分组成：
- 输入文件（视频、字幕）
- 滤镜链（字幕烧录）
- 输出参数（编码器、质量、格式）

### 3.2 字幕烧录

使用 FFmpeg 的 `subtitles` 或 `ass` 滤镜将字幕硬编码到视频中。

滤镜参数支持配置：
- 字体名称和大小
- 字体颜色和透明度
- 边框宽度和颜色
- 阴影效果
- 字幕位置（上/中/下）
- 背景色

### 3.3 双字幕叠加

支持同时烧录原始字幕和翻译字幕，两种字幕可配置不同的样式和位置：

**叠加方式**：
- 使用 FFmpeg 滤镜链依次叠加两个字幕轨道
- 先烧录第一层字幕，再烧录第二层字幕

**典型布局**：
- 翻译字幕在下方（主字幕），原始字幕在上方（副字幕）
- 或翻译字幕在下方，原始字幕在翻译字幕上方（紧邻）

**样式区分**：
- 主字幕（翻译）：较大字号，醒目颜色
- 副字幕（原始）：较小字号，次要颜色或半透明

### 3.4 视频编码设置

支持多种视频编码格式：

| 编码格式 | FFmpeg 编码器 | 硬件加速编码器 | 说明 |
|----------|---------------|----------------|------|
| H.264 | libx264 | h264_nvenc (NVIDIA) / h264_qsv (Intel) | 兼容性最好，推荐默认 |
| H.265/HEVC | libx265 | hevc_nvenc (NVIDIA) / hevc_qsv (Intel) | 压缩率更高，文件更小 |
| AV1 | libaom-av1 / libsvtav1 | av1_nvenc (NVIDIA) | 最新编码，压缩率最高 |

**编码参数说明**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 视频编码器 | libx264 | 视频编码格式 |
| 视频质量 | crf=23 | 质量因子（越小质量越高，H.264/H.265: 0-51，AV1: 0-63） |
| 预设 | medium | 编码速度预设（ultrafast/fast/medium/slow/veryslow） |
| 音频编码器 | aac | AAC 音频编码 |
| 音频比特率 | 320k | 音频比特率 |
| 输出格式 | mp4 | 容器格式 |

**编码格式选择建议**：
- H.264：兼容性最好，适合需要在各种设备播放的场景
- H.265：文件体积比 H.264 小约 50%，适合存储空间有限的场景
- AV1：压缩效率最高，但编码速度较慢，适合对文件大小要求严格的场景

## 4. 字幕样式配置

### 4.1 样式参数

每种字幕（原始/翻译）可独立配置以下样式参数：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| font_name | string | "Microsoft YaHei" | 字体名称 |
| font_size | int | 24 | 字体大小 |
| font_color | string | "#FFFFFF" | 字体颜色（十六进制） |
| font_alpha | float | 1.0 | 字体透明度（0-1） |
| outline_width | int | 2 | 边框宽度 |
| outline_color | string | "#000000" | 边框颜色 |
| shadow_enabled | bool | true | 是否启用阴影 |
| shadow_color | string | "#000000" | 阴影颜色 |
| shadow_offset | int | 2 | 阴影偏移 |
| position | string | "bottom" | 位置（top/center/bottom） |
| margin_v | int | 30 | 垂直边距 |
| margin_h | int | 20 | 水平边距 |
| background_enabled | bool | false | 是否启用背景色 |
| background_color | string | "#000000" | 背景颜色 |
| background_alpha | float | 0.5 | 背景透明度 |

### 4.2 双字幕样式配置

双字幕模式下，需要分别配置主字幕（翻译）和副字幕（原始）的样式：

**主字幕（翻译字幕）默认样式**：
- 位置：底部上方
- 字号：24
- 颜色：淡黄色（#FFFACD）
- 透明度：0.85（半透明）
- 边框：黑色，2px

**副字幕（原始字幕）默认样式**：
- 位置：底部（在翻译字幕下方）
- 字号：18
- 颜色：白色（#FFFFFF）
- 透明度：0.85（半透明）
- 边框：黑色，1px

### 4.3 自定义样式预设

用户可以创建和保存自定义样式预设，方便随时加载使用：

**预设管理功能**：
- 创建新预设：基于当前样式配置保存为新预设
- 加载预设：从已保存的预设列表中选择加载
- 编辑预设：修改已有预设的样式参数
- 删除预设：删除不需要的自定义预设
- 导入/导出：支持预设配置的导入导出

**预设存储**：
- 系统预设：内置的默认预设，不可删除
- 用户预设：用户自定义的预设，存储在数据库中

**预设配置结构**：
- 预设名称
- 预设描述
- 主字幕样式配置
- 副字幕样式配置（双字幕模式）
- 创建时间
- 更新时间

### 4.4 系统内置预设

系统提供几种内置预设样式供用户快速选择：

| 预设 | 说明 |
|------|------|
| default | 淡黄色翻译字幕 + 白色原始字幕，半透明，底部显示 |
| youtube | 类似 YouTube 自动字幕样式 |
| netflix | 类似 Netflix 字幕样式 |
| minimal | 简约风格，无边框阴影 |
| high_contrast | 高对比度，适合复杂背景 |
| single_translated | 仅翻译字幕，白色，底部居中 |
| single_original | 仅原始字幕，白色，底部居中 |

### 4.5 ASS 字幕格式

对于复杂样式需求，支持将 SRT 转换为 ASS 格式，ASS 格式支持更丰富的样式控制：
- 多种字体样式
- 渐变效果
- 动画效果
- 精确定位

## 5. 进度解析

### 5.1 FFmpeg 输出解析

FFmpeg 在处理过程中输出进度信息，格式如：

```
frame=  120 fps= 30 q=28.0 size=    1024kB time=00:00:04.00 bitrate=2097.2kbits/s speed=1.0x
```

从输出中提取：
- `time`：已处理时长
- `speed`：处理速度
- `size`：已输出大小

### 5.2 进度计算

```
进度 = (已处理时长 / 视频总时长) × 100
```

### 5.3 上报内容

进度百分比、已处理时长、总时长、处理速度、预计剩余时间。

## 6. 配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| subtitle_mode | string | dual | 字幕模式 |
| video_codec | string | h264 | 视频编码格式（h264/h265/av1） |
| video_crf | int | 23 | 视频质量因子 |
| video_preset | string | medium | 编码预设 |
| use_hardware_encoder | bool | true | 是否使用硬件编码器 |
| audio_codec | string | aac | 音频编码器 |
| audio_bitrate | string | 320k | 音频比特率 |
| output_format | string | mp4 | 输出格式 |
| primary_subtitle_style | object | default | 主字幕（翻译）样式 |
| secondary_subtitle_style | object | secondary_default | 副字幕（原始）样式 |
| ffmpeg_path | string | ffmpeg | FFmpeg 可执行文件路径 |
| hardware_accel | string | none | 硬件加速（none/cuda/vaapi/qsv） |

视频编码格式可选值：
- `h264`：H.264/AVC 编码
- `h265`：H.265/HEVC 编码
- `av1`：AV1 编码

字幕模式可选值：
- `original_only`：仅原始字幕
- `translated_only`：仅翻译字幕
- `dual`：双字幕叠加

## 7. 硬件加速

### 7.1 支持的加速方式

| 方式 | 说明 | 适用平台 |
|------|------|----------|
| none | 纯 CPU 编码 | 所有平台 |
| cuda | NVIDIA GPU 加速 | NVIDIA 显卡 |
| vaapi | VA-API 加速 | Linux + Intel/AMD |
| qsv | Intel Quick Sync | Intel CPU |

### 7.2 CUDA 加速

使用 NVIDIA GPU 进行视频编解码：

| 编码格式 | 解码器 | 编码器 |
|----------|--------|--------|
| H.264 | h264_cuvid | h264_nvenc |
| H.265 | hevc_cuvid | hevc_nvenc |
| AV1 | av1_cuvid | av1_nvenc |

硬件加速可显著提升处理速度，特别是对于高分辨率视频。

### 7.3 自动检测

系统启动时自动检测可用的硬件加速方式，用户也可手动指定。

## 8. 错误处理

### 8.1 常见错误

| 错误类型 | 原因 | 处理方式 |
|----------|------|----------|
| 视频文件不存在 | 文件被删除或路径错误 | 不可重试 |
| 字幕文件不存在 | 未完成翻译 | 不可重试 |
| FFmpeg 执行失败 | 命令错误或资源不足 | 检查日志 |
| 编码器不可用 | 缺少编码器或硬件不支持 | 回退到软件编码 |
| 磁盘空间不足 | 存储空间不够 | 不可重试，告警 |
| 输出文件损坏 | 编码过程中断 | 可重试 |

### 8.2 错误恢复策略

- FFmpeg 执行失败：检查错误日志，尝试简化命令重试
- 硬件加速失败：自动回退到 CPU 编码
- 输出文件损坏：删除损坏文件，重新执行

## 9. 接口设计

### 9.1 服务接口

| 方法 | 说明 |
|------|------|
| execute(task, progress_callback) | 执行合成任务 |
| cancel(task_id) | 取消合成 |
| can_retry(error) | 判断错误是否可重试 |
| build_command(config) | 构建 FFmpeg 命令 |
| parse_progress(output) | 解析进度输出 |
| validate_output(file_path) | 验证输出文件 |

### 9.2 任务 Payload

| 字段 | 说明 |
|------|------|
| video_id | 视频 UUID |
| video_path | 原视频文件路径 |
| subtitle_mode | 字幕模式 |
| original_subtitle_path | 原始字幕文件路径（可选） |
| translated_subtitle_path | 翻译字幕文件路径（可选） |
| primary_subtitle_style | 主字幕样式配置 |
| secondary_subtitle_style | 副字幕样式配置（双字幕模式） |

### 9.3 任务 Result

| 字段 | 说明 |
|------|------|
| video_id | 视频 UUID |
| output_path | 输出视频路径 |
| duration | 视频时长 |
| file_size | 文件大小 |
| subtitle_mode | 使用的字幕模式 |
| hardware_accel | 使用的硬件加速 |

- 工作进程：调用执行合成任务
