# 数据库管理器详细设计文档

## 1. 模块概述

### 1.1 功能定位

数据库管理器是 SRT Flow 系统的核心基础模块，负责统一管理 SQLite 数据库的连接、会话和 ORM 映射。它为其他模块提供数据持久化服务，确保数据访问的一致性和可靠性。

### 1.2 核心职责

- **连接管理**：管理 SQLite 数据库连接，支持异步操作
- **会话管理**：提供数据库会话的创建、获取和生命周期管理
- **ORM 映射**：定义所有数据库表的 SQLAlchemy 模型
- **数据库初始化**：自动创建表结构，支持版本迁移
- **事务管理**：提供事务控制，确保数据一致性

### 1.3 设计原则

- **单一数据源**：所有数据库操作通过统一入口
- **异步优先**：使用 SQLAlchemy 2.0 异步 API
- **连接复用**：合理管理连接，避免资源泄漏
- **类型安全**：使用 SQLAlchemy ORM 确保类型正确

## 2. 数据库表结构设计

### 2.1 表结构总览

| 表名 | 说明 | 主要用途 |
|------|------|----------|
| videos | 视频信息 | 存储视频元数据 |
| tasks | 任务队列 | 存储任务状态和进度 |
| video_files | 视频文件 | 记录视频关联的各类文件 |
| tags | 标签 | 视频标签定义 |
| video_tags | 视频标签关联 | 多对多关联表 |

| config | 系统配置 | 存储用户配置 |
| task_logs | 任务日志 | 存储任务执行日志 |

### 2.2 videos 表

存储视频的元数据信息。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键，UUID |
| title | VARCHAR(500) | 视频标题 |
| source | VARCHAR(50) | 来源：youtube/bilibili/local |
| external_id | VARCHAR(100) | 外部平台 ID（用于去重） |
| channel_name | VARCHAR(200) | 频道/UP主名称 |
| channel_id | VARCHAR(100) | 频道 ID |
| duration | INTEGER | 时长（秒） |
| thumbnail_url | TEXT | 原始缩略图 URL |
| description | TEXT | 视频描述 |
| directory_path | VARCHAR(500) | 本地存储目录 |
| group_id | INTEGER | 所属分组（外键） |
| metadata | JSON | 扩展元数据 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

唯一约束：(source, external_id)

索引：source, created_at, group_id

### 2.3 tasks 表

存储所有异步任务的状态和进度。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键，UUID |
| video_id | VARCHAR(36) | 关联视频（外键，可为空） |
| type | VARCHAR(50) | 任务类型 |
| status | VARCHAR(20) | 状态：pending/running/completed/failed/cancelled |
| progress | INTEGER | 进度 0-100 |
| priority | VARCHAR(10) | 优先级：high/normal/low |
| payload | JSON | 任务参数 |
| result | JSON | 执行结果 |
| error | TEXT | 错误信息 |
| retry_count | INTEGER | 已重试次数 |
| max_retries | INTEGER | 最大重试次数，默认 3 |
| created_at | TIMESTAMP | 创建时间 |
| started_at | TIMESTAMP | 开始执行时间 |
| completed_at | TIMESTAMP | 完成时间 |

索引：status, type, video_id, (priority, created_at)

### 2.4 video_files 表

记录视频目录下的各类文件。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| video_id | VARCHAR(36) | 关联视频（外键） |
| file_type | VARCHAR(50) | 文件类型 |
| file_path | VARCHAR(500) | 相对路径 |
| file_size | INTEGER | 文件大小（字节） |

- video_original：原始视频

## 3. 数据库管理器设计

### 3.1 单例模式

数据库管理器采用单例模式，确保全局唯一实例，避免重复创建连接。

### 3.2 生命周期管理

**初始化阶段**：
1. 创建异步数据库引擎（aiosqlite）
2. 配置连接池（SQLite 使用 StaticPool）
3. 创建会话工厂
4. 执行表结构创建（如不存在）
5. 执行数据库迁移（如需要）

**运行阶段**：
- 通过会话工厂获取数据库会话
- 会话使用上下文管理器自动提交/回滚
- 支持显式事务控制

**关闭阶段**：
- 释放所有连接
- 关闭数据库引擎

### 3.3 会话管理

提供两种会话获取方式：

1. **上下文管理器**：自动管理事务，异常时回滚
2. **依赖注入**：用于 FastAPI 路由，请求结束自动关闭

### 3.4 SQLite 特殊配置

- 启用 WAL 模式提升并发性能
- 设置 `check_same_thread=False` 支持多线程
- 使用 StaticPool 适配异步场景

## 4. Repository 模式

### 4.1 设计思路

为每个实体提供 Repository 类，封装常用的数据访问操作，业务层通过 Repository 访问数据，不直接操作 ORM 模型。

### 4.2 通用 Repository 方法

| 方法 | 说明 |
|------|------|
| get_by_id(id) | 根据 ID 获取单个实体 |
| get_all(limit, offset) | 获取所有实体（分页） |
| create(entity) | 创建实体 |
| update(entity) | 更新实体 |
| delete(entity) | 删除实体 |
| count() | 获取实体总数 |

### 4.3 VideoRepository 特有方法

| 方法 | 说明 |
|------|------|
| get_by_external_id(source, external_id) | 根据来源和外部 ID 查找（去重用） |
| search(keyword, source, tag_ids, group_id) | 搜索视频 |
| get_with_files(video_id) | 获取视频及其关联文件 |

### 4.4 TaskRepository 特有方法

| 方法 | 说明 |
|------|------|
| get_pending_tasks(task_type, limit) | 获取待执行任务 |
| get_running_tasks(task_type) | 获取正在执行的任务 |
| get_by_video_id(video_id) | 获取视频关联的所有任务 |
| update_status(task_id, status, progress, error, result) | 更新任务状态 |
| count_by_status() | 统计各状态任务数量 |

## 5. 数据库迁移

### 5.1 迁移策略

由于项目使用 SQLite 且为单用户工具，采用简化的迁移策略：

1. 在 config 表中记录数据库版本号
2. 应用启动时检查版本并执行必要迁移
3. 新增字段使用默认值，避免破坏性变更

### 5.2 迁移流程

1. 读取当前数据库版本
2. 对比目标版本
3. 依次执行中间版本的迁移脚本
4. 更新版本号
