# 下载服务详细设计文档

## 1. 模块概述

### 1.1 功能定位

下载服务负责从 YouTube、Bilibili 等平台下载视频，是 SRT Flow 处理流水线的第一步。它封装了 yt-dlp 和 BiliDown 等下载工具，提供统一的下载接口。

### 1.2 核心职责

- **视频下载**：从支持的平台下载视频文件
- **元数据提取**：获取视频标题、时长、上传者等信息
- **去重检查**：避免重复下载已存在的视频
- **目录管理**：创建规范的视频存储目录结构
- **进度上报**：实时报告下载进度

### 1.3 下载通道

系统提供两种下载通道，由用户自由选择：

| 通道 | 支持平台 | 说明 |
|------|----------|------|
| yt-dlp | YouTube、Bilibili、其他 | 通用下载工具，支持大多数平台 |
| BiliDown | Bilibili | Bilibili 专用工具，支持高清下载 |

**通道选择规则**：
- 用户在提交下载任务时选择下载通道
- 系统根据 URL 识别平台，与用户选择的通道进行核验
- 如果通道不支持该平台，返回错误提示

## 2. 业务流程

### 2.1 下载流程

```
用户提交 URL + 选择下载通道
    │
    ▼
解析 URL，识别平台
    │
    ▼
核验通道与平台兼容性
    │
    ├── 不兼容 ──→ 返回错误，提示选择正确通道
    │
    ▼
去重检查（source + external_id）
    │
    ├── 已存在 ──→ 返回已有视频信息
    │
    ▼
创建视频目录
    │
    ▼
调用选定的下载通道
    │
    ▼
解析元数据
    │
    ▼
保存视频记录到数据库
    │
    ▼
返回下载结果
```

### 2.2 目录结构

每个视频创建独立目录：

```
data/downloads/<UUID>_<SafeTitle>/
├── video.mp4              # 原始视频
├── video.info.json        # yt-dlp 输出的元数据
├── audio_original.m4a     # 提取的音频（可选）
└── task.log               # 任务日志
```

目录命名规则：
- UUID：系统生成的唯一标识
- SafeTitle：视频标题的安全版本（特殊字符替换为下划线，限制长度）

## 3. 下载工具封装

### 3.1 yt-dlp 封装

yt-dlp 是主要的下载工具，支持大多数视频平台。

**调用方式**：通过子进程调用命令行工具

**核心参数**：

| 参数 | 说明 |
|------|------|
| -f | 视频格式选择，默认 "bestvideo+bestaudio/best" |
| -o | 输出文件路径模板 |
| --write-info-json | 输出元数据 JSON 文件 |
| --write-thumbnail | 下载缩略图 |
| --write-subs | 下载字幕（如有） |
| --proxy | 代理设置 |
| --cookies-from-browser | 从浏览器读取 cookies |
| --progress | 输出进度信息 |

**H264 编码过滤**：

当 `force_h264=true` 时，使用 `[vcodec~='^(h264|avc)']` 过滤器优先选择 H264 编码的视频流。这是因为 HEVC (H265) 编码在大多数浏览器中不支持原生播放，会导致视频预览功能无法使用。

格式选择策略：
- 优先选择 H264 编码的最佳质量
- 如果没有 H264 格式，回退到其他可用编码
- 默认开启，确保浏览器兼容性

**进度解析**：
- 解析 yt-dlp 的标准输出
- 提取下载百分比、速度、剩余时间
- 转换为 0-100 的进度值上报

### 3.2 BiliDown 封装

Bilibili 专用下载工具，作为独立的下载通道供用户选择。

**特点**：
- 支持登录下载会员视频
- 支持高清晰度视频下载
- 针对 Bilibili 优化

**调用方式**：通过子进程调用命令行工具

### 3.3 通道核验策略

1. 用户选择下载通道（yt-dlp 或 BiliDown）
2. 系统解析 URL 识别平台
3. 核验通道与平台的兼容性：
   - yt-dlp：支持 YouTube、Bilibili 及其他平台
   - BiliDown：仅支持 Bilibili
4. 如果不兼容，返回错误提示用户选择正确的通道
5. 记录使用的通道到日志

## 4. 去重检查

### 4.1 去重标识

使用 `source + external_id` 作为唯一标识：

| 平台 | source | external_id 示例 |
|------|--------|------------------|
| YouTube | youtube | dQw4w9WgXcQ |
| Bilibili | bilibili | BV1xx411c7mD |

### 4.2 去重流程

1. 从 URL 提取平台和视频 ID
2. 查询数据库是否存在相同记录
3. 如果存在：
   - 返回已有视频信息
   - 可选：提示用户是否强制重新下载
4. 如果不存在：继续下载流程

### 4.3 URL 解析

支持多种 URL 格式：

**YouTube**：
- `https://www.youtube.com/watch?v=VIDEO_ID`
- `https://youtu.be/VIDEO_ID`
- `https://www.youtube.com/shorts/VIDEO_ID`

**Bilibili**：
- `https://www.bilibili.com/video/BV1xxxxxx`
- `https://b23.tv/xxxxxx`

## 5. 元数据提取

### 5.1 元数据来源

yt-dlp 下载时生成 `video.info.json`，包含丰富的元数据。

### 5.2 提取的字段

| 字段 | 来源字段 | 说明 |
|------|----------|------|
| title | title | 视频标题 |
| duration | duration | 时长（秒） |
| channel_name | uploader / channel | 上传者/频道名 |
| channel_id | uploader_id / channel_id | 频道 ID |
| description | description | 视频描述 |
| thumbnail_url | thumbnail | 缩略图 URL |
| upload_date | upload_date | 上传日期 |
| view_count | view_count | 播放量 |
| like_count | like_count | 点赞数 |

### 5.3 元数据存储

- 原始 JSON 保存到 `video.info.json`
- 关键字段提取后存入数据库 videos 表
- 扩展字段存入 metadata JSON 字段

## 6. 配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| preferred_quality | string | best | 首选视频质量 |
| download_subtitles | bool | true | 是否下载平台字幕 |
| download_thumbnail | bool | true | 是否下载缩略图 |
| proxy_url | string | null | 代理地址 |
| cookies_browser | string | null | 读取 cookies 的浏览器 |
| max_filesize | string | null | 最大文件大小限制 |
| allowed_time_start | string | null | 允许下载时段开始 |
| allowed_time_end | string | null | 允许下载时段结束 |
| yt_dlp_path | string | yt-dlp | yt-dlp 可执行文件路径 |
| bilidown_path | string | null | BiliDown 可执行文件路径 |

### 6.1 视频质量选项

| 选项 | 说明 |
|------|------|
| best | 最佳质量（视频+音频） |
| 1080p | 1080p 或更低 |
| 720p | 720p 或更低 |
| 480p | 480p 或更低 |
| audio_only | 仅音频 |

## 7. 错误处理

### 7.1 常见错误

| 错误类型 | 原因 | 处理方式 |
|----------|------|----------|
| 网络错误 | 连接超时、DNS 失败 | 可重试 |
| 视频不存在 | URL 无效或视频已删除 | 不可重试 |
| 需要登录 | 会员视频、年龄限制 | 提示配置 cookies |
| 地区限制 | 视频在当前地区不可用 | 提示配置代理 |
| 格式不可用 | 请求的质量不存在 | 降级到可用质量 |
| 磁盘空间不足 | 存储空间不够 | 不可重试，告警 |

### 7.2 错误恢复

- 网络错误：自动重试，最多 3 次
- 部分下载：yt-dlp 支持断点续传
- 通道不兼容：提示用户选择正确的下载通道

## 8. 进度上报

### 8.1 进度计算

下载进度直接从 yt-dlp 输出解析：

```
[download]  45.2% of 125.00MiB at 2.50MiB/s ETA 00:25
```

解析后上报：
- progress: 45
- speed: "2.50MiB/s"
- eta: "00:25"

### 8.2 多文件下载

如果下载包含视频和音频分离的格式：
- 视频下载占 60%
- 音频下载占 30%
- 合并处理占 10%

## 9. 接口设计

### 9.1 服务接口

| 方法 | 说明 |
|------|------|
| execute(task, progress_callback) | 执行下载任务 |
| cancel(task_id) | 取消下载 |
| can_retry(error) | 判断错误是否可重试 |
| parse_url(url) | 解析 URL，返回平台和视频 ID |
| validate_channel(channel, platform) | 核验通道与平台的兼容性 |
| check_duplicate(source, external_id) | 检查是否重复 |

### 9.2 任务 Payload

```json
{
  "url": "https://www.youtube.com/watch?v=xxx",
  "channel": "yt-dlp",
  "quality": "best",
  "download_subtitles": true,
  "force_download": false,
  "force_h264": true
}
```

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| url | string | - | 视频 URL（必填） |
| channel | string | yt-dlp | 下载通道 |
| quality | string | best | 视频质量 |
| download_subtitles | bool | true | 是否下载字幕 |
| force_download | bool | false | 强制重新下载 |
| force_h264 | bool | true | 强制 H264 编码（确保浏览器预览兼容） |

### 9.3 任务 Result

```json
{
  "video_id": "uuid",
  "directory_path": "data/downloads/uuid_title/",
  "files": {
    "video": "video.mp4",
    "info": "video.info.json",
    "thumbnail": "thumbnail.jpg"
  },
  "metadata": {
    "title": "Video Title",
    "duration": 360,
    "channel_name": "Channel"
  }
}
```

## 10. 依赖关系

下载服务依赖：
- 配置管理器：读取下载配置
- 数据库管理器：存储视频记录
- 日志管理器：记录下载日志
- 外部工具：yt-dlp、BiliDown

被以下模块依赖：
- 工作进程：调用执行下载任务
- 媒体管理服务：查询视频信息
