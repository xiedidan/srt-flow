# 语音合成服务详细设计文档

## 1. 模块概述

### 1.1 功能定位

语音合成服务（TTS）负责将翻译后的字幕转换为语音，是 SRT Flow 处理流水线的可选环节。它封装了多种本地 TTS 引擎，提供统一的语音合成接口，支持语速计算和 GPU 加速。

### 1.2 核心职责

- **字幕解析**：解析翻译后的 SRT 字幕文件
- **句子合并**：使用 AI 将分散的字幕片段合并为完整句子（可选）
- **语速计算**：根据字幕时长和文本长度计算合适的语速
- **长文本切分**：将长字幕切分为适合 TTS 处理的片段
- **语音合成**：调用 TTS 引擎生成语音
- **音频拼接**：将多段语音拼接为完整音频
- **进度上报**：实时报告合成进度

### 1.3 支持的 TTS 引擎

#### 本地引擎

| 引擎 | 特点 | 语言支持 |
|------|------|----------|
| Coqui TTS | 开源，质量好 | 多语言 |
| ChatTTS | 自然度高，支持情感 | 中文、英文 |
| SparkTTS | 讯飞开源，中文优秀 | 中文 |
| IndexTTS | 零样本克隆 | 中文、英文 |
| CozyVoice | 阿里开源，多风格 | 中文、英文 |
| VITS 系 | 轻量快速 | 多语言 |

#### 在线服务

| 引擎 | 特点 | 语言支持 |
|------|------|----------|
| Azure TTS | 微软翻译器 TTS，免费，支持情感风格 | 多语言 |
| Edge TTS | 微软 Edge 浏览器 TTS，免费，稳定 | 多语言 |
| 火山 TTS | 字节跳动翻译 TTS，免费，中文优秀 | 中文、日语、英语 |

## 2. 业务流程

### 2.1 合成流程

```
接收合成任务
    │
    ▼
检查翻译字幕文件是否存在
    │
    ├── 不存在 ──→ 返回错误
    │
    ▼
解析 SRT 字幕文件
    │
    ▼
句子合并（如启用）
    │
    ▼
加载 TTS 引擎（如未加载）
    │
    ▼
┌─────────────────────────┐
│  循环处理每条字幕        │
│    │                    │
│    ▼                    │
│  计算目标语速           │
│    │                    │
│    ▼                    │
│  长文本切分（如需要）    │
│    │                    │
│    ▼                    │
│  调用 TTS 引擎合成      │
│    │                    │
│    ▼                    │
│  调整音频时长           │
│    │                    │
│    ▼                    │
│  保存临时音频片段       │
│    │                    │
│    ▼                    │
│  更新进度               │
└─────────────────────────┘
    │
    ▼
拼接所有音频片段
    │
    ▼
输出最终音频文件
    │
    ▼
清理临时文件
    │
    ▼
更新数据库记录
    │
    ▼
返回合成结果
```

### 2.2 输出文件

合成完成后在视频目录生成 `audio_tts.m4a`（合成的语音文件）。

## 3. TTS 引擎封装

### 3.1 统一接口

所有 TTS 引擎实现统一接口，包含以下方法：
- **load_model**：加载 TTS 模型
- **synthesize**：合成语音，接收文本、语速、说话人参数，返回音频数据
- **get_sample_rate**：获取音频采样率
- **unload_model**：卸载模型释放内存

### 3.2 引擎特点

**Coqui TTS**：开源社区维护，支持多种语言和模型，可通过模型名称加载不同的预训练模型。

**ChatTTS**：支持情感控制和说话人嵌入，生成的语音自然度高，适合对话场景。

**SparkTTS**：讯飞开源的中文 TTS 引擎，中文发音质量优秀。

**IndexTTS**：支持零样本语音克隆，可通过参考音频生成相似音色。

**CozyVoice**：阿里开源，支持多种说话风格和情感。

**VITS 系**：轻量级端到端 TTS，推理速度快，适合实时场景。

### 3.3 引擎管理

引擎管理器负责：
- 按需创建和缓存引擎实例
- 避免重复加载同一引擎
- 提供引擎切换能力

## 4. 句子合并（AI 辅助）

### 4.1 功能说明

由于字幕有长度限制，很多句子会从中间分开，导致语音合成时不连贯。句子合并功能使用 AI 将分散的字幕片段智能合并为完整句子，提升语音的自然度和流畅性。

### 4.2 两阶段处理架构

为了支持超长视频（5000+ 条字幕），采用两阶段分批处理架构：

**阶段 1：AI 判断合并关系**
- 将字幕条目文本发送给 AI
- AI 只返回分组关系（如 `1,2,3|4,5|6`），不返回合并后的文本
- 输出格式极简，token 消耗少，不会超长

**阶段 2：本地合并处理**
- 根据 AI 返回的分组关系，在本地将字幕条目分组
- 每组内的字幕文本拼接起来
- 分批进行 TTS 合成（纯技术性分批，不涉及语义判断）

### 4.3 自动分批策略

**单批次处理**（< 4000 条字幕）：
- 一次性发送所有字幕给 AI 判断
- AI 看到完整上下文，合并判断最准确
- 适用于大多数视频（1-3 小时）

**多批次处理**（≥ 4000 条字幕）：
- 在标点符号处智能切分成多个批次
- 每批次独立调用 AI 判断合并关系
- 本地汇总所有分组关系后统一处理
- 适用于超长视频（4+ 小时）

### 4.4 智能切分逻辑

切分优先级：
1. 句子结束标点（。！？.!?）
2. 逗号等次级标点（，,）
3. 理想位置（如无标点）

切分窗口：在理想切分点前后 ±100 条字幕范围内搜索标点。

### 4.5 Token 容量分析

- DeepSeek V3/R1 上下文：128K tokens
- 单条字幕平均：约 20 tokens（含序号和格式）
- 可处理字幕数：128K ÷ 20 ≈ 6,400 条
- 保守阈值：4,000 条（预留 system prompt 和输出空间）

### 4.6 配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| enable_sentence_merge | bool | false | 是否启用句子合并 |
| sentence_merge_provider_id | string | null | AI 服务商 ID |
| sentence_merge_model | string | deepseek-chat | 模型名称 |
| sentence_merge_temperature | float | 0.3 | 温度参数（0-1） |
| sentence_merge_system_prompt | string | null | 自定义系统提示词 |
| sentence_merge_user_prompt | string | null | 自定义用户提示词 |

### 4.7 优势

1. **避免断句**：AI 看到完整上下文，不存在批次衔接问题
2. **输出极简**：只返回数字分组，token 消耗极少
3. **逻辑简单**：本地分批是纯机械操作，不会出错
4. **结果准确**：AI 能看到所有上下文，合并判断最准确
5. **可扩展**：支持任意长度的字幕文件

## 5. 语速计算

### 4.1 目标语速计算

根据字幕的可用时长和文本长度计算目标语速：

1. 计算文本字符数
2. 根据基准语速（默认 4 字符/秒）计算自然朗读时长
3. 目标语速 = 自然时长 / 可用时长
4. 将语速限制在合理范围（0.5 - 2.0 倍）

### 5.2 语速探测

合成前可先用样本文本探测引擎的实际语速，获取该引擎在 1.0 倍速下的实际字符/秒速率，用于更精确的语速计算。

### 5.3 时长调整

如果语速调整后仍无法匹配目标时长，可使用音频时间拉伸技术（如 librosa 的 time_stretch）进行微调，但应尽量避免过度拉伸影响音质。

## 6. 长文本切分

### 6.1 切分原因

TTS 引擎通常对单次输入长度有限制，过长的文本可能导致：
- 合成失败
- 音质下降
- 内存溢出

### 6.2 切分策略

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| max_chars | 100 | 单次合成最大字符数 |
| split_punctuation | 。！？，、；：.!?,;: | 优先切分的标点 |

切分规则：
1. 如果文本长度不超过限制，直接合成
2. 超过限制时，优先在句子边界（句号、问号等）切分
3. 无合适标点时，在逗号等次级标点处切分
4. 切分后的每个片段独立合成，然后拼接

### 6.3 时长分配

切分后，将字幕的总时长按片段长度比例分配给各片段，确保整体时长匹配。

## 7. 音频处理

### 7.1 音频拼接

将所有字幕对应的音频片段按时间顺序拼接：
1. 创建与视频等长的空白音频
2. 将每条字幕的音频放置到对应的时间位置
3. 字幕之间的空白保持静音

### 7.2 输出格式

| 格式 | 编码 | 说明 |
|------|------|------|
| WAV | PCM | 无损，文件大 |
| M4A | AAC | 有损压缩，推荐 |
| MP3 | MP3 | 兼容性好 |

默认输出 M4A 格式（AAC 编码，128k 比特率），使用 FFmpeg 进行格式转换。

## 8. GPU 加速

### 8.1 GPU 检测

系统启动时自动检测 CUDA GPU 是否可用，并获取显存大小信息。

### 8.2 设备选择

- 优先使用 GPU（如果可用且配置启用）
- GPU 不可用时使用 CPU
- 支持指定 GPU 设备 ID（多卡环境）

### 8.3 内存管理

合成完成后清理 GPU 显存缓存，避免内存泄漏。

## 10. 进度计算

### 10.1 进度公式

```
进度 = (已合成字幕数 / 总字幕数) × 100
```

### 10.2 上报内容

进度百分比、当前处理的字幕序号、总字幕数、当前使用的引擎、状态消息。

## 11. 错误处理

### 11.1 常见错误

| 错误类型 | 原因 | 处理方式 |
|----------|------|----------|
| 字幕文件不存在 | 文件被删除或路径错误 | 不可重试 |
| 模型加载失败 | 模型文件损坏或内存不足 | 尝试其他引擎 |
| GPU 内存不足 | 显存不够 | 任务失败，提示用户 |
| 合成失败 | 引擎内部错误 | 可重试 |
| 音频处理失败 | FFmpeg 错误 | 检查安装 |

### 11.2 错误恢复策略

- GPU 内存不足：任务失败，提示用户释放显存或使用 CPU 模式
- 单条合成失败：跳过并记录，继续处理其他字幕
- 引擎加载失败：尝试备选引擎

## 12. 接口设计

### 12.1 服务接口

| 方法 | 说明 |
|------|------|
| execute(task, progress_callback) | 执行合成任务 |
| cancel(task_id) | 取消合成 |
| can_retry(error) | 判断错误是否可重试 |
| load_engine(engine_name) | 加载 TTS 引擎 |
| synthesize_subtitle(entry, config) | 合成单条字幕 |
| concatenate_segments(segments) | 拼接音频片段 |

### 12.2 任务 Payload

| 字段 | 说明 |
|------|------|
| video_id | 视频 UUID |
| subtitle_path | 翻译后字幕文件路径 |
| engine | TTS 引擎名称 |
| speaker | 说话人/音色（或参考音频文件路径） |
| reference_audio_id | 参考音频 ID（可选，用于语音克隆） |
| output_format | 输出格式 |

### 12.3 任务 Result

| 字段 | 说明 |
|------|------|
| video_id | 视频 UUID |
| subtitle_path | 输入字幕路径 |
| output_path | 输出音频路径 |
| engine_used | 使用的引擎 |
| total_entries | 总字幕条数 |
| duration | 音频时长（秒） |
| sample_rate | 采样率 |
