# 媒体管理服务详细设计文档

## 1. 模块概述

### 1.1 功能定位

媒体管理服务负责管理系统中所有视频及其关联文件，提供文件索引、标签管理、分组管理等功能。它是用户组织和查找视频资源的核心模块。

### 1.2 核心职责

- **文件索引**：扫描和索引视频目录下的所有文件
- **元数据管理**：维护视频的元数据信息
- **标签管理**：支持为视频添加、删除、查询标签
- **分组管理**：支持用户自定义视频分组
- **文件关联**：管理视频与各类文件的关联关系
- **搜索查询**：提供多维度的视频搜索功能

### 1.3 设计原则

- **数据一致性**：文件系统与数据库保持同步
- **灵活组织**：支持多种方式组织视频
- **高效查询**：优化搜索和过滤性能

## 2. 文件索引方案

### 2.1 索引范围

扫描 `data/downloads/` 目录下的所有视频目录，索引以下文件：

| 文件类型 | 文件名模式 | 说明 |
|----------|------------|------|
| video_original | video.mp4 | 原始视频 |
| audio_original | audio_original.m4a | 提取的音频 |
| subtitle_original | subtitle_original.srt | 原始字幕 |
| subtitle_translated | subtitle_translated.srt | 翻译字幕 |
| audio_tts | audio_tts.m4a | 合成语音 |
| video_output | video_output.mp4 | 输出视频 |
| thumbnail | thumbnail.jpg | 缩略图 |
| summary | assets/summary.txt | 摘要 |
| info | video.info.json | 元数据 |

### 2.2 索引流程

**初始化索引**：
1. 扫描 downloads 目录下所有子目录
2. 对每个目录检查是否有 video.info.json
3. 解析元数据，创建或更新视频记录
4. 扫描目录内文件，更新 video_files 表

**增量索引**：
1. 监听文件系统变化（可选）
2. 定期扫描检查新增/删除的文件
3. 更新数据库记录

### 2.3 文件状态同步

处理文件系统与数据库不一致的情况：

| 场景 | 处理方式 |
|------|----------|
| 数据库有记录但文件不存在 | 标记文件为缺失或删除记录 |
| 文件存在但数据库无记录 | 创建新记录 |
| 文件大小/校验和变化 | 更新文件记录 |

## 3. 标签系统设计

### 3.1 标签类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 系统标签 | 自动生成，不可删除 | 来源:youtube、状态:已翻译 |
| 用户标签 | 用户创建和管理 | 教程、音乐、待处理 |
| 任务组标签 | 批量任务自动创建 | batch-20241215 |

### 3.2 自动标签规则

系统根据视频属性自动添加标签：

| 条件 | 自动标签 |
|------|----------|
| source = youtube | 来源:YouTube |
| source = bilibili | 来源:Bilibili |
| 存在 subtitle_translated | 状态:已翻译 |
| 存在 audio_tts | 状态:已配音 |
| 存在 video_output | 状态:已合成 |
| duration > 3600 | 时长:长视频 |
| duration < 60 | 时长:短视频 |

### 3.3 标签操作

| 操作 | 说明 |
|------|------|
| 创建标签 | 指定名称和颜色 |
| 删除标签 | 同时移除所有视频的该标签 |
| 重命名标签 | 更新标签名称 |
| 添加到视频 | 为视频添加标签 |
| 从视频移除 | 移除视频的标签 |
| 批量操作 | 为多个视频添加/移除标签 |

### 3.4 标签颜色

预设颜色方案：

| 颜色名 | HEX 值 | 用途建议 |
|--------|--------|----------|
| 蓝色 | #3B82F6 | 默认 |
| 绿色 | #22C55E | 已完成 |
| 黄色 | #EAB308 | 进行中 |
| 红色 | #EF4444 | 重要/问题 |
| 紫色 | #A855F7 | 分类 |
| 灰色 | #6B7280 | 归档 |

## 4. 分组管理

### 4.1 分组概念

分组是用户组织视频的方式，一个视频只能属于一个分组（或不属于任何分组）。

### 4.2 分组与标签的区别

| 特性 | 分组 | 标签 |
|------|------|------|
| 数量关系 | 一对多（视频只能属于一个分组） | 多对多 |
| 用途 | 互斥分类 | 多维度标记 |
| 示例 | 项目A、项目B | 教程、已翻译、重要 |

### 4.3 分组操作

| 操作 | 说明 |
|------|------|
| 创建分组 | 指定名称和描述 |
| 删除分组 | 分组内视频变为未分组 |
| 重命名分组 | 更新分组名称 |
| 移动视频 | 将视频移入/移出分组 |
| 批量移动 | 批量移动多个视频 |

### 4.4 默认分组

系统预设分组（不可删除）：

| 分组 | 说明 |
|------|------|
| 未分组 | 虚拟分组，表示不属于任何分组的视频 |
| 回收站 | 标记删除的视频（可恢复） |

## 5. 搜索与过滤

### 5.1 搜索维度

| 维度 | 说明 | 示例 |
|------|------|------|
| 关键词 | 标题模糊匹配 | "Python 教程" |
| 来源 | 视频平台 | youtube, bilibili |
| 标签 | 包含指定标签 | 已翻译, 教程 |
| 分组 | 属于指定分组 | 项目A |
| 时长范围 | 视频时长 | 5-30 分钟 |
| 日期范围 | 创建/下载时间 | 最近7天 |
| 状态 | 处理状态 | 有字幕, 有配音 |

### 5.2 排序选项

| 排序字段 | 说明 |
|----------|------|
| created_at | 创建时间（默认，降序） |
| title | 标题字母序 |
| duration | 时长 |
| channel_name | 频道名 |

### 5.3 分页

- 默认每页 20 条
- 支持自定义每页数量（最大 100）
- 返回总数用于前端分页

## 6. 文件关联管理

### 6.1 关联文件类型

每个视频可关联多种类型的文件，每种类型最多一个文件：

| 类型 | 说明 | 来源 |
|------|------|------|
| video_original | 原始视频 | 下载服务 |
| audio_original | 原始音频 | ASR 服务提取 |
| subtitle_original | 原始字幕 | ASR 服务生成 |
| subtitle_translated | 翻译字幕 | 翻译服务生成 |
| audio_tts | 合成语音 | TTS 服务生成 |
| video_output | 输出视频 | 视频合成服务 |
| thumbnail | 缩略图 | 下载或生成 |
| summary | 摘要 | 素材生成服务 |

### 6.2 文件操作

| 操作 | 说明 |
|------|------|
| 查看文件列表 | 获取视频关联的所有文件 |
| 读取文件内容 | 读取文本文件（字幕、摘要） |
| 更新文件内容 | 保存编辑后的文本文件 |
| 关联外部文件 | 用户上传或指定外部文件 |
| 删除文件 | 删除指定类型的文件 |

### 6.3 用户文件关联

支持用户关联自行处理的文件：
- 用户可上传自己制作的字幕文件
- 用户可指定外部处理的视频文件
- 关联后更新 video_files 记录

## 7. 视频生命周期

### 7.1 状态流转

```
下载中 → 已下载 → 处理中 → 已完成
                      ↓
                   处理失败
```

### 7.2 删除策略

**软删除**：
1. 将视频移入回收站分组
2. 保留文件和数据库记录
3. 可随时恢复

**硬删除**：
1. 从回收站彻底删除
2. 删除视频目录及所有文件
3. 删除数据库记录
4. 不可恢复

### 7.3 回收站管理

- 回收站视频保留 30 天
- 超期自动硬删除
- 支持手动清空回收站

## 8. 接口设计

### 8.1 视频管理接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /videos | GET | 获取视频列表（支持搜索过滤） |
| /videos/{id} | GET | 获取视频详情 |
| /videos/{id} | PATCH | 更新视频信息 |
| /videos/{id} | DELETE | 删除视频（软删除） |
| /videos/{id}/files | GET | 获取视频文件列表 |
| /videos/{id}/files/{type}/content | GET | 读取文件内容 |
| /videos/{id}/files/{type}/content | PUT | 保存文件内容 |

### 8.2 标签管理接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /tags | GET | 获取所有标签 |
| /tags | POST | 创建标签 |
| /tags/{id} | PATCH | 更新标签 |
| /tags/{id} | DELETE | 删除标签 |
| /videos/{id}/tags | POST | 为视频添加标签 |
| /videos/{id}/tags/{tag_id} | DELETE | 移除视频标签 |

### 8.3 分组管理接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /groups | GET | 获取所有分组 |
| /groups | POST | 创建分组 |
| /groups/{id} | PATCH | 更新分组 |
| /groups/{id} | DELETE | 删除分组 |
| /videos/{id}/group | PUT | 设置视频分组 |

## 9. 配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| downloads_path | path | data/downloads | 视频存储根目录 |
| trash_retention_days | int | 30 | 回收站保留天数 |
| auto_index_interval | int | 3600 | 自动索引间隔（秒） |
| enable_file_watch | bool | false | 启用文件系统监听 |
| max_search_results | int | 1000 | 搜索最大返回数 |

## 10. 依赖关系

媒体管理服务依赖：
- 数据库管理器：存储视频、标签、分组数据
- 配置管理器：读取配置
- 日志管理器：记录操作日志

被以下模块依赖：
- API 层：提供视频管理接口
- 下载服务：创建视频记录
- 其他业务服务：查询视频信息
