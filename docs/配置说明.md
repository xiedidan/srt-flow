# SRT Flow 配置说明

本文档详细说明 SRT Flow 的各项配置选项。

## 1. 环境变量配置

环境变量通过 `.env` 文件或 Docker 环境变量设置。

### 1.1 服务器配置

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `HOST` | string | `0.0.0.0` | 服务监听地址 |
| `PORT` | int | `8010` | 服务端口 |
| `DEBUG` | bool | `false` | 调试模式，开启后显示详细错误 |

### 1.2 数据库配置

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `DATABASE_PATH` | string | `data/srtflow.db` | SQLite 数据库文件路径 |

### 1.3 文件存储配置

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `DATA_DIR` | string | `data` | 数据根目录 |
| `DOWNLOADS_DIR` | string | `data/downloads` | 视频下载目录 |
| `LOG_DIR` | string | `data/logs` | 日志文件目录 |

### 1.4 安全配置

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `SECRET_KEY` | string | 必填 | 会话密钥，用于签名 |
| `ENCRYPTION_KEY` | string | 必填 | AES-256 加密密钥，用于加密 API Key |

**生成密钥：**

```bash
# 生成 64 字符的十六进制密钥
python -c "import secrets; print(secrets.token_hex(32))"
```

### 1.5 日志配置

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `LOG_LEVEL` | string | `INFO` | 日志级别：DEBUG/INFO/WARNING/ERROR |

### 1.6 外部工具配置

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `FFMPEG_PATH` | string | `ffmpeg` | FFmpeg 可执行文件路径 |
| `YTDLP_PATH` | string | `yt-dlp` | yt-dlp 可执行文件路径 |

### 1.7 GPU 配置

| 变量 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `CUDA_VISIBLE_DEVICES` | string | `0` | 可见的 GPU 设备，多个用逗号分隔 |

## 2. 功能配置（Web 界面）

以下配置通过 Web 界面的「设置」页面进行管理，存储在数据库中。

### 2.1 全局配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 默认源语言 | string | `en` | 默认源语言 |
| 默认目标语言 | string | `zh-CN` | 默认目标语言 |
| 日志级别 | string | `INFO` | 日志级别：DEBUG/INFO/WARNING/ERROR |
| 最大并发任务数 | int | `1` | 同时执行的最大任务数 |
| 代理地址 | string | 空 | 全局代理，用于视频下载、模型下载、API 调用等 |

**代理配置说明：**

代理地址是全局配置，适用于：
- 视频下载（yt-dlp）
- Whisper 模型下载（Hugging Face）
- LLM API 调用（翻译服务等）

格式示例：`http://127.0.0.1:7890` 或 `socks5://127.0.0.1:1080`

### 2.2 下载服务配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 视频质量 | string | `best` | 下载视频质量 |
| 下载字幕 | bool | `true` | 是否下载视频自带字幕 |
| 允许下载开始时间 | string | 空 | 允许下载的开始时间（HH:MM） |
| 允许下载结束时间 | string | 空 | 允许下载的结束时间（HH:MM） |

### 2.3 语音识别配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 识别引擎 | string | `whisper_local` | 引擎类型：whisper_local/faster_whisper_xxl/gemini |
| 模型大小 | string | `medium` | Whisper 模型：tiny/base/small/medium/large/large-v2/large-v3 |
| 设备 | string | `auto` | 运行设备：auto/cuda/cpu |
| 额外参数 | string | 空 | Whisper 命令行额外参数 |
| Gemini API 地址 | string | 空 | 自定义 Gemini API 地址，留空使用官方默认地址 |
| Gemini API Key | string | 空 | Gemini API 密钥（加密存储） |

**模型与引擎下载说明：**

- 模型文件：跨平台共享，从 Hugging Face 下载 faster-whisper 模型，两种本地引擎共用
- 两种本地引擎都需下载平台特定的可执行文件（Windows/Linux/macOS）
- 文件存储在 `data/models/` 目录
- 支持下载进度显示，可随时删除已下载的模型或引擎文件

**Gemini API 地址说明：**

支持使用第三方 Gemini API 代理服务，配置时填写完整的 API 地址（如 `https://api.example.com/v1beta`），系统会自动拼接对应的端点。留空则使用官方默认地址。

### 2.4 翻译服务配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 提供商 | string | `deepseek` | LLM 提供商：deepseek/gemini/openai/anthropic/openai_compatible |
| 模型名称 | string | 按提供商 | 使用的模型名称 |
| API 地址 | string | 空 | 自定义 API 地址，留空使用官方默认地址 |
| API Key | string | 必填 | API 密钥（加密存储） |
| 目标语言 | string | `zh` | 翻译目标语言 |
| 批次大小 | int | `30` | 每批翻译的字幕条数 |
| 批次上下文行数 | int | `3` | 每批次前后包含的上下文行数（0-10），用于提高翻译连贯性 |
| 温度 | float | `0.3` | 生成温度 |

**批次上下文说明：**

为了让翻译批次之间衔接更流畅，可以配置批次上下文行数：
- 每个批次翻译时，会将前后 N 行内容作为上下文一并提交给 AI
- 上下文行仅供参考，AI 只翻译当前批次的内容
- 增加重叠可以提高翻译连贯性，但会增加 API 调用成本
- 创建翻译任务时可以覆盖此默认值

**API 地址说明：**

所有 LLM 提供商都支持自定义 API 地址，适用于：
- 使用第三方 API 代理服务
- 本地部署的 LLM（如 Ollama、vLLM）
- 企业内网部署的 API 服务

配置时填写完整的 API 地址（如 `https://api.example.com/v1`），系统会自动拼接对应的端点。留空则使用官方默认地址。

### 2.5 语音合成配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| TTS 引擎 | string | `coqui` | 引擎：coqui/chattts/vits |
| 语音模型 | string | 按引擎 | 使用的语音模型 |
| 输出格式 | string | `m4a` | 输出音频格式：wav/m4a |
| 语速 | float | `1.0` | 语音速度倍率 |

### 2.6 视频合成配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 字幕字体 | string | `Noto Sans CJK` | 字幕字体 |
| 字幕大小 | int | `24` | 字幕字号 |
| 字幕颜色 | string | `#FFFFFF` | 字幕颜色 |
| 字幕位置 | string | `bottom` | 位置：top/center/bottom |
| 边框宽度 | int | `2` | 字幕边框宽度 |
| 边框颜色 | string | `#000000` | 边框颜色 |
| 替换音轨 | bool | `false` | 是否用 TTS 替换原音轨 |

### 2.7 素材生成配置

素材生成包含三个子任务（摘要、标题、缩略图），具体生成哪些在新建任务时选择，此处配置各子任务的 AI 参数。

#### 摘要生成配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| AI 服务商 | string | 空 | 选择已配置的 AI 服务商 |
| 模型名称 | string | `deepseek-chat` | 使用的模型名称 |
| 温度 | float | `0.65` | 生成温度（较低以获得更准确的摘要） |

#### 标题生成配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 候选数量 | int | `3` | 生成的标题候选数量 |
| AI 服务商 | string | 空 | 选择已配置的 AI 服务商 |
| 模型名称 | string | `deepseek-chat` | 使用的模型名称 |
| 温度 | float | `0.85` | 生成温度（较高以获得更有创意的标题） |

#### 缩略图 AI 生成配置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| AI 服务商 | string | 空 | 选择已配置的 AI 服务商 |
| 模型名称 | string | `deepseek-chat` | 使用的模型名称 |
| 温度 | float | `0.7` | 生成温度 |

**注意**：新建任务时如果选择 AI 生成缩略图，但未配置 AI 服务商，系统会提示用户先完成配置。

### 2.8 任务时段配置

每个服务可独立配置允许执行的时段：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 启用时段控制 | bool | `false` | 是否启用 |
| 开始时间 | string | `00:00` | 允许执行的开始时间 |
| 结束时间 | string | `23:59` | 允许执行的结束时间 |

## 3. 配置文件示例

### .env 文件

```bash
# 服务器
HOST=0.0.0.0
PORT=8010
DEBUG=false

# 数据库
DATABASE_PATH=data/srtflow.db

# 文件存储
DATA_DIR=data
DOWNLOADS_DIR=data/downloads
LOG_DIR=data/logs

# 安全（请替换为随机生成的值）
SECRET_KEY=your-64-char-hex-secret-key
ENCRYPTION_KEY=your-64-char-hex-encryption-key

# 日志
LOG_LEVEL=INFO

# GPU
CUDA_VISIBLE_DEVICES=0
```

## 4. Docker 配置

### docker-compose.yml 环境变量

```yaml
environment:
  - HOST=0.0.0.0
  - PORT=8010
  - DEBUG=false
  - DATABASE_PATH=/app/data/srtflow.db
  - DATA_DIR=/app/data
  - DOWNLOADS_DIR=/app/data/downloads
  - LOG_DIR=/app/data/logs
  - LOG_LEVEL=INFO
```

### 数据卷挂载

```yaml
volumes:
  - ./data:/app/data          # 数据持久化
  - ./.env:/app/.env:ro       # 环境变量文件
```

## 5. 安全建议

1. **密钥管理**
   - 生产环境必须使用随机生成的 SECRET_KEY 和 ENCRYPTION_KEY
   - 不要将密钥提交到版本控制

2. **API Key 保护**
   - API Key 在数据库中加密存储
   - 前端无法查看 API Key 明文
   - 定期轮换 API Key

3. **网络安全**
   - 生产环境建议使用反向代理（Nginx）
   - 启用 HTTPS
   - 限制访问 IP

4. **数据备份**
   - 定期备份 `data/` 目录
   - 包括数据库和视频文件
