# 工作进程详细设计文档

## 1. 模块概述

### 1.1 功能定位

工作进程（Worker）是 SRT Flow 任务调度系统的执行单元，负责从队列中获取任务并分发给对应的业务服务执行。它是连接队列管理器和业务服务的桥梁。

### 1.2 核心职责

- **任务消费**：从队列管理器获取待执行任务
- **任务分发**：根据任务类型调用对应的业务服务
- **生命周期管理**：管理任务的启动、执行、完成/失败流程
- **并发控制**：确保每种任务类型最多一个并发
- **进度上报**：将任务进度实时更新到队列
- **异常处理**：捕获执行异常，记录错误信息

### 1.3 设计原则

- **单一职责**：Worker 只负责调度，不包含业务逻辑
- **松耦合**：通过统一接口与业务服务交互
- **可观测**：完整的日志记录和状态上报
- **优雅停止**：支持平滑关闭，等待当前任务完成

```
        │          │          │          │

```

### 2.2 组件说明

| 组件 | 说明 |
|------|------|
| WorkerManager | 管理所有 Worker 的生命周期 |
| Worker | 单个任务类型的消费者 |
| QueueManager | 任务队列，提供任务获取和状态更新 |
| Service | 具体业务服务，执行实际任务逻辑 |

### 2.3 Worker 与任务类型对应关系

每种任务类型对应一个独立的 Worker 实例：

| Worker | 任务类型 | 对应服务 |
|--------|----------|----------|
| DownloadWorker | download | DownloadService |
| ASRWorker | asr | ASRService |
| TranslateWorker | translate | TranslateService |
| TTSWorker | tts | TTSService |
| SynthesizeWorker | synthesize | VideoSynthesisService |
| AssetGenWorker | asset_gen | AssetGenerationService |

## 3. 并发控制策略

### 3.1 并发规则

1. **类型内串行**：同一类型的任务串行执行，最多 1 个并发
2. **类型间并行**：不同类型的任务可以并行执行（流水线）
3. **资源隔离**：每个 Worker 独立运行，互不影响

### 3.2 并发控制实现

- 每个 Worker 独立轮询自己类型的任务
- 出队时检查是否已有该类型的 running 任务
- 如果有，则跳过本次轮询，等待下一周期

### 3.3 流水线效果

由于不同类型可并行，实现流水线处理效果：

```
时间轴 →
Video A: [download] → [asr] → [translate] → [tts] → [synthesize]
Video B:            [download] → [asr] → [translate] → [tts] → ...
Video C:                      [download] → [asr] → ...
```

## 4. 任务执行流程

### 4.1 主循环流程

1. **检查停止信号**：如果收到停止信号，退出循环
2. **获取任务**：从队列获取该类型的待执行任务
3. **无任务处理**：如果没有任务，等待一段时间后继续
4. **执行任务**：调用业务服务执行任务
5. **更新状态**：根据执行结果更新任务状态
6. **继续循环**：返回步骤 1

### 4.2 任务执行详细流程

```
获取任务
    │
    ▼
设置任务上下文（日志分流）
    │
    ▼
更新状态为 running
    │
    ▼
调用业务服务执行
    │
    ├── 成功 ──→ 更新状态为 completed，保存结果
    │
    └── 失败 ──→ 记录错误
                    │
                    ├── 可重试 ──→ 更新状态为 failed，等待调度器重试
                    │
                    └── 不可重试 ──→ 更新状态为 failed，标记为最终失败
```

### 4.3 进度上报机制

业务服务通过回调函数上报进度：

1. 业务服务执行过程中调用进度回调
2. Worker 接收进度值（0-100）
3. Worker 调用队列管理器更新进度
4. 队列管理器触发事件通知前端

进度上报频率控制：
- 最小间隔 1 秒，避免频繁更新
- 进度变化超过 5% 时立即上报

## 5. WorkerManager 设计

### 5.1 职责

- 创建和管理所有 Worker 实例
- 统一启动和停止所有 Worker
- 监控 Worker 健康状态
- 处理 Worker 异常重启

### 5.2 生命周期管理

**启动流程**：
1. 读取配置，确定需要启动的 Worker 类型
2. 为每种任务类型创建 Worker 实例
3. 启动所有 Worker（异步任务）
4. 注册信号处理器（SIGTERM、SIGINT）

**停止流程**：
1. 收到停止信号
2. 设置停止标志
3. 等待所有 Worker 完成当前任务
4. 超时强制终止（默认 30 秒）
5. 清理资源

### 5.3 健康检查

定期检查 Worker 状态：
- Worker 是否存活
- 最后活动时间
- 当前执行的任务

异常处理：
- Worker 意外退出时自动重启
- 连续失败超过阈值时告警

## 6. 业务服务接口规范

### 6.1 统一接口

所有业务服务必须实现统一的执行接口：

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| execute | task, progress_callback | result | 执行任务 |
| cancel | task_id | bool | 取消任务 |
| can_retry | error | bool | 判断错误是否可重试 |

### 6.2 执行方法规范

**输入参数**：
- task：任务对象，包含 id、type、payload 等
- progress_callback：进度回调函数，接受 0-100 的整数

**返回值**：
- 成功：返回结果字典，存入 task.result
- 失败：抛出异常，异常信息存入 task.error

### 6.3 可重试错误判断

业务服务需要判断错误是否可重试：

| 可重试 | 不可重试 |
|--------|----------|
| 网络超时 | 参数错误 |
| API 限流 | 文件不存在 |
| 临时服务不可用 | 认证失败 |
| 资源暂时不足 | 格式不支持 |

## 7. 异常处理

### 7.1 异常分类

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| 业务异常 | 任务执行失败 | 记录错误，标记任务失败 |
| 系统异常 | Worker 内部错误 | 记录日志，重启 Worker |
| 资源异常 | 内存/磁盘不足 | 暂停 Worker，告警 |

### 7.2 错误恢复

**任务级恢复**：
- 可重试错误：等待调度器安排重试
- 不可重试错误：标记为最终失败

**Worker 级恢复**：
- 单次异常：记录日志，继续运行
- 连续异常：重启 Worker
- 持续异常：停止 Worker，告警

### 7.3 系统重启恢复

系统重启时，WorkerManager 执行恢复流程：
1. 扫描 running 状态的任务
2. 根据任务类型判断是否可安全重试
3. 可重试任务重置为 pending
4. 不可重试任务标记为 failed

## 8. 配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| poll_interval | float | 1.0 | 轮询间隔（秒） |
| shutdown_timeout | int | 30 | 优雅停止超时（秒） |
| max_consecutive_errors | int | 5 | 最大连续错误次数 |
| progress_report_interval | float | 1.0 | 进度上报最小间隔（秒） |
| progress_report_threshold | int | 5 | 进度变化上报阈值（%） |
| enabled_workers | list | 全部 | 启用的 Worker 类型列表 |

## 9. 事件通知

Worker 在关键节点发出事件：

| 事件 | 触发时机 | 用途 |
|------|----------|------|
| worker.started | Worker 启动 | 监控 |
| worker.stopped | Worker 停止 | 监控 |
| worker.task_started | 开始执行任务 | 日志、通知 |
| worker.task_completed | 任务完成 | 触发下游任务 |
| worker.task_failed | 任务失败 | 告警、重试调度 |
| worker.error | Worker 异常 | 告警、自动恢复 |

## 10. 依赖关系

工作进程依赖：
- 队列管理器：获取任务、更新状态
- 日志管理器：记录执行日志
- 配置管理器：读取配置
- 各业务服务：执行具体任务

被以下模块依赖：
- 应用入口：启动 WorkerManager
