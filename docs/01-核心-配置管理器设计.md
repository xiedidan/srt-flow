# 配置管理器详细设计文档

## 1. 模块概述

### 1.1 功能定位

配置管理器是 SRT Flow 系统的核心基础模块，负责统一管理系统的各类配置信息。它为其他模块提供配置读写服务，确保配置的一致性和安全性。

### 1.2 核心职责

- **配置分层管理**：管理系统级配置（.env）和用户级配置（SQLite）
- **敏感信息保护**：API Key 等敏感信息的加密存储与脱敏展示
- **配置验证**：在配置写入前进行格式和有效性验证
- **配置热加载**：支持运行时配置更新（部分配置无需重启）
- **默认值管理**：为各配置项提供合理的默认值

### 1.3 设计原则

- **安全优先**：敏感信息必须加密存储，前端不可查询明文
- **单一职责**：仅负责配置的读写和管理
- **易于扩展**：新增配置项时无需修改核心逻辑
- **类型安全**：使用 Pydantic 进行配置验证

## 2. 配置分层设计

### 2.1 配置层次结构

系统采用三层配置架构，从底层到顶层依次为：

1. **代码默认值**：硬编码在配置模型中，作为兜底
2. **全局配置**：存储在 SQLite，项目共用设置
3. **功能配置**：存储在 SQLite，各服务独立配置
4. **环境变量**：.env 文件，部署相关配置，优先级最高

### 2.2 配置类型说明

| 配置类型 | 存储位置 | 说明 | 示例 |
|---------|---------|------|------|
| 系统级配置 | `.env` 文件 | 部署相关、不常变更 | 数据库路径、端口号、加密密钥 |
| 全局配置 | SQLite | 项目共用配置 | 默认语言、日志级别 |
| 功能配置 | SQLite | 各功能独立配置 | TTS 引擎选择、LLM Provider |

### 2.3 配置优先级

配置读取时按以下顺序查找，找到即返回：
环境变量 > 功能配置 > 全局配置 > 代码默认值

## 3. 数据结构设计

### 3.1 系统配置（.env）

系统级配置通过环境变量加载，包含以下字段：

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| DATABASE_PATH | path | data/srt_flow.db | 数据库文件路径 |
| DOWNLOAD_PATH | path | data/downloads | 视频下载目录 |
| LOG_PATH | path | data/logs | 日志文件目录 |
| SERVER_HOST | string | 0.0.0.0 | 服务监听地址 |
| SERVER_PORT | int | 8010 | 服务端口 |
| ENCRYPTION_KEY | string | 自动生成 | 敏感信息加密密钥 |

### 3.2 SQLite 配置表

配置表采用 Key-Value 结构存储，支持分类管理：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| category | VARCHAR(50) | 配置类别：global/download/asr/tts/translate 等 |
| key | VARCHAR(100) | 配置键名 |
| value | TEXT | 配置值（JSON 序列化） |
| is_encrypted | BOOLEAN | 是否加密存储 |
| description | TEXT | 配置说明 |
| updated_at | TIMESTAMP | 更新时间 |

唯一约束：(category, key)

### 3.3 功能配置分类

| 类别 | 说明 | 主要配置项 |
|------|------|-----------|
| global | 全局配置 | 默认语言、日志级别、最大并发数 |
| download | 下载服务 | 视频质量、代理地址、执行时段 |
| asr | 语音识别 | 引擎选择、模型大小、计算设备、Gemini AI 服务商 |
| translate | 翻译服务 | AI 服务商、模型名称、批次大小、提示词 |
| tts | 语音合成 | TTS 引擎、输出格式、语速 |
| synthesis | 视频合成 | 字幕样式、音量设置 |
| asset | 素材生成 | AI 服务商、生成选项、缩略图方式 |

### 3.4 AI 服务商表 (ai_providers)

独立存储 AI API 配置，供各服务引用：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | UUID 主键 |
| name | VARCHAR(100) | 用户自定义名称 |
| api_type | VARCHAR(50) | API 类型：deepseek/openai/gemini/openai_compatible |
| base_url | TEXT | API 地址（可选，有默认值） |
| api_key | TEXT | 加密存储的 API Key |
| is_enabled | BOOLEAN | 是否启用 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

各服务配置通过 `ai_provider_id` 字段引用 AI 服务商。

## 4. 敏感信息加密方案

### 4.1 加密算法

采用 AES-256-GCM 对称加密：
- 提供足够的安全强度
- 同时提供加密和完整性校验
- 适合单用户本地部署场景

### 4.2 密钥管理

加密密钥的获取优先级：
1. 环境变量 ENCRYPTION_KEY（推荐生产环境使用）
2. 本地密钥文件 .encryption_key（首次启动自动生成）

密钥派生使用 PBKDF2-SHA256，迭代 100000 次。

### 4.3 加密存储流程

1. 前端提交敏感配置（如 API Key）
2. 配置管理器使用 AES-256-GCM 加密
3. 加密后的密文存入数据库，标记 is_encrypted=true
4. 读取时自动解密供内部服务使用

### 4.4 脱敏显示

前端查询敏感配置时，返回脱敏格式：
- 原值：`sk-abcdefghijklmnop`
- 脱敏：`sk-a***`

规则：保留前 4 个字符，其余用 `*` 替代。

## 5. 接口设计

### 5.1 内部接口

| 方法 | 说明 |
|------|------|
| get_system_settings() | 获取系统配置（.env） |
| get_config(category) | 获取指定类别的配置 |
| update_config(category, updates) | 更新配置（部分更新） |
| reset_config(category) | 重置为默认值 |
| get_api_key(provider) | 获取 API Key 明文（仅内部调用） |
| set_api_key(provider, key) | 设置 API Key（加密存储） |
| get_masked_api_key(provider) | 获取脱敏的 API Key |
| validate_config(category, config) | 验证配置有效性 |
| export_config() | 导出配置（排除敏感信息） |
| import_config(data) | 导入配置 |

### 5.2 REST API

| 端点 | 方法 | 说明 |
|------|------|------|
| /api/v1/config/{category} | GET | 获取配置（敏感字段脱敏） |
| /api/v1/config/{category} | PUT | 更新配置 |
| /api/v1/config/{category}/secrets/{key} | PUT | 设置敏感配置 |
| /api/v1/config/{category}/validate | POST | 验证配置 |
| /api/v1/config/{category}/test | POST | 测试连接（如验证 API Key） |

## 6. 边界情况处理

### 6.1 首次启动

- 自动创建配置表结构
- 自动生成加密密钥并保存到文件
- 使用默认值初始化必要配置

### 6.2 加密密钥变更

- 检测到密钥变更时记录警告日志
- 无法解密的旧配置返回错误，提示用户重新配置

### 6.3 配置迁移

- 版本升级时自动检测配置结构变化
- 新增字段使用默认值填充
- 废弃字段保留但不再读取

---

## 7. 错误处理

| 异常类型 | 说明 |
|----------|------|
| ConfigNotFoundError | 配置不存在 |
| ConfigValidationError | 配置验证失败 |
| ConfigDecryptionError | 配置解密失败 |
| ConfigEncryptionError | 配置加密失败 |

## 8. 依赖关系

配置管理器依赖：
- 数据库管理器：存储配置数据
- 日志管理器：记录配置变更

被以下模块依赖：
- 所有业务服务（下载、ASR、TTS、翻译等）
- 队列管理器
- 调度器
