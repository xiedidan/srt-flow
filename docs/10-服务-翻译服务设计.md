# 翻译服务详细设计文档

## 1. 模块概述

### 1.1 功能定位

翻译服务负责将语音识别生成的原始字幕翻译为目标语言，是 SRT Flow 处理流水线的核心环节。它封装了多种 LLM API，提供统一的翻译接口，支持批次处理和进度追踪。

### 1.2 核心职责

- **字幕解析**：解析 SRT 格式字幕文件
- **批次处理**：将字幕分批发送给 LLM 翻译
- **LLM 调用**：封装多种 LLM API（DeepSeek、Gemini、OpenAI）
- **故障转移**：批次失败时按优先级尝试下一个 LLM 实例
- **结果重组**：将翻译结果重组为 SRT 格式
- **进度上报**：基于批次计算并上报翻译进度

### 1.3 支持的 LLM 提供商

| 提供商 | 模型 | 说明 |
|--------|------|------|
| DeepSeek | DeepSeek R1 | 性价比高，中文翻译质量好 |
| Google | Gemini Pro | 多语言支持好 |
| OpenAI | GPT-4 / GPT-3.5 | 翻译质量稳定 |
| OpenAI Compatible | 自定义 | 兼容 OpenAI API 格式的任意服务 |

### 1.4 多实例配置

系统支持配置多个 LLM 实例，同一提供商可配置多个不同的 API 地址和 Key：

- 用户可配置多个 LLM 实例，按优先级排序
- 批次翻译失败时，按配置顺序尝试下一个实例
- 直到批次成功或所有实例都失败
- 不支持降级到单条翻译

## 2. 业务流程

### 2.1 翻译流程

```
接收翻译任务
    │
    ▼
检查原始字幕文件是否存在
    │
    ├── 不存在 ──→ 返回错误
    │
    ▼
解析 SRT 字幕文件
    │
    ▼
按配置分批（如每 30 条一批）
    │
    ▼
┌─────────────────────────────────┐
│  循环处理每个批次               │
│    │                           │
│    ▼                           │
│  构建翻译 Prompt               │
│    │                           │
│    ▼                           │
│  调用 LLM API（带故障转移）     │
│    │                           │
│    ▼                           │
│  解析翻译结果                  │
│    │                           │
│    ▼                           │
│  更新进度                      │
└─────────────────────────────────┘
    │
    ▼
合并所有批次结果
    │
    ▼
生成翻译后的 SRT 文件
    │
    ▼
更新数据库记录
    │
    ▼
返回翻译结果
```

### 2.2 输出文件

翻译完成后在视频目录生成 `subtitle_translated.srt`。

## 3. SRT 字幕解析

### 3.1 SRT 格式结构

标准 SRT 格式包含：序号、时间轴、字幕文本（可多行），各条目以空行分隔。

### 3.2 解析规则

- 按空行分隔字幕条目
- 第一行为序号（数字）
- 第二行为时间轴（包含 `-->`）
- 后续行为字幕文本（可能多行）
- 忽略空白条目

### 3.3 解析结果

每条字幕解析为包含序号、开始时间、结束时间、文本的数据结构。

## 4. 批次处理

### 4.1 分批策略

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| batch_size | 30 | 每批字幕条数 |
| max_tokens_per_batch | 5000 | 每批最大 token 数 |
| context_overlap_lines | 3 | 批次上下文行数（前后各 N 行作为上下文）|

分批规则：
1. 优先按条数分批（默认 30 条）
2. 如果单批 token 数超限，减少条数
3. 确保不在句子中间断开

### 4.2 批次上下文（Context Overlap）

为了让翻译批次之间衔接更流畅，支持批次上下文功能：

- 每个批次翻译时，会将前后 N 行内容作为上下文一并提交给 AI
- 上下文行标记为 `[CONTEXT]`，明确指示 AI 只翻译标记为 `[TRANSLATE]` 的行
- 重叠行数可在全局配置中设置默认值，也可在创建任务时覆盖
- 增加重叠可以提高翻译连贯性，但会增加 API 调用成本

### 4.3 批次故障转移

批次翻译失败时的处理流程：

1. 在当前 LLM 实例重试（指数退避，最多重试配置的次数）
2. 重试失败后，按优先级尝试下一个 LLM 实例
3. 所有实例都失败则批次失败，任务终止
4. **不支持降级到单条翻译**

## 5. LLM 实例配置

### 5.1 实例配置项

每个 LLM 实例包含以下配置：

| 配置项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| name | string | 是 | 实例名称（用于日志和显示） |
| provider | string | 是 | 提供商类型：deepseek/gemini/openai/openai_compatible |
| api_key | string | 是 | API Key（加密存储） |
| base_url | string | 是 | API 地址（openai_compatible 必填，其他可选） |
| model | string | 是 | 模型名称 |
| temperature | float | 否 | 温度参数，默认 0.3 |
| enabled | bool | 否 | 是否启用，默认 true |
| priority | int | 否 | 优先级（数字越小优先级越高），默认 100 |

### 5.2 配置示例

可配置多个实例，例如：
- DeepSeek 官方（优先级 1）
- DeepSeek 代理（优先级 2）
- OpenAI GPT-4（优先级 3）

系统按优先级顺序尝试，直到翻译成功。

### 5.3 实例管理

LLM 实例管理器负责：
- 加载并按优先级排序所有启用的实例
- 为每个实例创建对应的 Provider
- 提供按优先级顺序获取 Provider 的接口

## 6. Prompt 设计

### 6.1 翻译 Prompt 要点

Prompt 包含以下要素：
- 角色设定：专业字幕翻译专家
- 源语言和目标语言
- 翻译要求：保持语气风格、自然流畅、简洁适合视频观看
- 输出格式：按编号输出，每行一条

### 6.2 上下文提供

可选提供视频上下文信息以提高翻译质量：
- 视频标题
- 视频描述
- 频道/UP主名称
- 视频类型（教程、娱乐、新闻等）

### 6.3 任务背景（Task Context）

用户可在创建翻译任务时提供任务背景信息，帮助 AI 更准确地翻译：
- 专业术语解释
- 人名、地名等专有名词
- 特定语境说明
- 翻译风格要求

任务背景通过 `{task_context}` 占位符注入到用户提示词的开头，格式为：
```
Background:
<用户提供的背景信息>

Video context:
...
```

### 6.4 输出格式

要求 LLM 按编号格式输出（如 `1. 翻译结果`），便于解析和验证数量。

## 7. 结果解析与重组

### 7.1 解析 LLM 输出

从 LLM 返回的文本中提取翻译结果：
- 按行分割
- 移除编号前缀（如 `1. ` 或 `1: `）
- 验证结果数量是否与原文匹配
- 数量不匹配则抛出错误，触发重试或故障转移

### 7.2 重组 SRT 文件

将翻译结果与原字幕的时间轴信息合并，生成新的 SRT 文件：
- 保持原有序号
- 保持原有时间轴
- 替换文本为翻译结果

## 8. 进度计算

### 8.1 进度公式

```
进度 = (已完成批次数 / 总批次数) × 100
```

### 8.2 上报内容

进度百分比、当前批次、总批次数、已翻译条数、总条数、状态消息。

## 10. 错误处理

### 10.1 常见错误

| 错误类型 | 原因 | 处理方式 |
|----------|------|----------|
| 字幕文件不存在 | 文件被删除或路径错误 | 不可重试 |
| SRT 解析失败 | 格式错误 | 不可重试 |
| API 调用失败 | 网络错误或服务不可用 | 尝试下一个 LLM 实例 |
| API 配额用尽 | 超出使用限制 | 尝试下一个 LLM 实例 |
| 翻译结果数量不匹配 | LLM 输出格式异常 | 重试当前实例或尝试下一个 |
| API Key 无效 | 配置错误 | 尝试下一个 LLM 实例 |
| 所有实例失败 | 所有配置的 LLM 都失败 | 任务失败，不可重试 |

### 10.2 错误恢复策略

- **单实例失败**：自动尝试下一个 LLM 实例（按优先级顺序）
- **网络错误**：在当前实例重试（指数退避），失败后尝试下一个实例
- **结果不匹配**：重试当前实例，失败后尝试下一个实例
- **所有实例失败**：批次标记为失败，任务终止
- **部分批次成功**：保存已翻译部分，记录失败批次位置，支持断点续传

## 11. 接口设计

### 11.1 服务接口

| 方法 | 说明 |
|------|------|
| execute(task, progress_callback) | 执行翻译任务 |
| cancel(task_id) | 取消翻译 |
| can_retry(error) | 判断错误是否可重试 |
| parse_srt(file_path) | 解析 SRT 文件 |
| translate_batch(entries, config) | 翻译一批字幕 |
| generate_srt(entries, translations) | 生成 SRT 文件 |

### 11.2 任务 Payload

| 字段 | 说明 |
|------|------|
| video_id | 视频 UUID |
| subtitle_path | 原始字幕文件路径 |
| source_language | 源语言 |
| target_language | 目标语言 |
| batch_size | 每批条数 |
| context_overlap_lines | 批次上下文行数（覆盖配置默认值）|
| resume_from_batch | 断点续传起始批次 |
| task_context | 任务背景信息（用户提供的上下文，帮助 AI 理解翻译内容）|

### 11.3 任务 Result

| 字段 | 说明 |
|------|------|
| video_id | 视频 UUID |
| source_path | 原始字幕路径 |
| output_path | 翻译后字幕路径 |
| source_language | 源语言 |
| target_language | 目标语言 |
| total_entries | 总字幕条数 |
| batches_processed | 处理的批次数 |
| providers_used | 使用的 LLM 实例列表 |
| context_overlap_lines | 实际使用的批次上下文行数 |
| prompt_info | 提示词信息（包含 system_prompt、user_prompt_template、task_context、context_overlap_lines）|

## 12. 依赖关系

翻译服务依赖：
- 配置管理器：读取翻译配置和 API Key
- 数据库管理器：更新视频记录
- 日志管理器：记录翻译日志
- 语音识别服务：提供原始字幕文件

被以下模块依赖：
- 工作进程：调用执行翻译任务
- 语音合成服务：使用翻译后的字幕
- 视频合成服务：使用翻译后的字幕
- 素材生成服务：使用翻译后的字幕生成摘要
